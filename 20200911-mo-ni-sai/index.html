F4的第n场

wzoi

题目背景

bleaves最近在wzoi上面做题。

wzoi的题目有两种，一种是noip题，一种是省选题。

bleaves的做题方式很特别。每一天，她可能会看一道题目，这时她会选择题目种类，然后wzoi会在选定种类中随机扔给她一道她还没看过的题，她会把这道题看一遍，然后存在脑子里慢慢思考；她也有可能写题，这时她一定会写没写过的题中看的时间最迟的一题(如果不存在没写过的且没看过的题，她就不能写题)。

题目描述

wzoi每天会有一个推荐的题目种类,

如果bleaves看一道题目：如果种类和推荐的相同，那么这道题目最大得分为10，否则为5

如果bleaves写一道题目：如果种类和推荐的相同，那么这道题目得分为最大得分，否则为最大得分-5

假如bleaves现在还没看过任何一题，并且她知道了wzoi接下来一些天每天推荐的种类，问她在这些天的最大得分。

输入输出格式

输入格式

一行一个01串s，|s|表示天数，${s_i}$=0表示wzoi第i天推荐noip题，${s_i}$=1表示wzoi第i天推荐省选题。

输出格式

一行一个整数最大得分。

输入输出样例

输入样例 #1

输出样例 #1

输入样例 #2

输出样例 #2

输入样例 #3

输出样例 #3

说明

样例1解释

4天行动依次为：

看一道noip题，写第1天看的题，看一道省选题，写第3天看的题。

数据范围：

全部的输入数据满足：1<=n<=10^6, n为偶数。

测试点1：n=2

测试点2：n=4

测试点3：n<=10

测试点4：n<=20

测试点5：n<=100

测试点6：n<=1000

测试点7、8、9、10：n<=10^6

思路

首先对于确定的一天看题一天做题，如果两天的推荐相同，那么收益是十，如果不同，那么是五。

所以如果安排好了看题和做题那么只需要累加贡献就行。

看题和做题的匹配像是括号的匹配，如果知道了这个括号序列那就可以用栈统计贡献，并且每个括号序列的答案是一定的。

所以我们有了一个 $\mathcal{O}(n\cdot2^n)$ 的做法，枚举括号序列然后 $\mathcal{O}(n)$ 计算贡献。如果遇到栈空情况下的右括号不合法，直接退出。最后输出个最大值。勉强能拿 $40$ 分。

然后我们再想怎么优化这一过程。

这个过程中有大量的冗余信息，包括右括号单独出现的不合法情况，还有很多左括号不能匹配白白浪费了一堆得分机会的情况。

后面的 $60$ 分BlueQuantum神仙给我两分钟就讲明白了。

其实根本不用考虑出来所有情况，只贪心地取就好了。

课程(course)

题目背景

bleaves最近在自学大学课程。

共有n个课程，每个课程可能有一个前驱课程，表示要学这个课程必须先学前驱课程。

题目描述

bleaves自学完一个课程就会把课程名字的首字母写下来，学完所有课程后所有字母构成了一个字符串，字母顺序就是她学的顺序。

bleaves有一些讨厌的单词，她不希望最终的字符串中出现这些单词。

现在，bleaves会从所有可能的课程顺序中选一种出来按该顺序自学，她想知道最终字符串不出现她讨厌的单词的概率。

输入输出格式

输入格式

第一行一个正整数n，表示bleaves要自学的课程数。

接下来一行n个整数，第i个整数表示第i个课程的前驱课程，如果为0则表示没有。

接下来一行一个长度为n的仅含小写字母的字符串，第i个字符表示第i个课程名字的首字母。

接下来一行一个正整数m，表示bleaves讨厌的单词数。

接下来一行m个仅含小写字母的字符串，表示bleaves讨厌的单词，保证长度不超过20。

输出格式

一行一个实数表示答案。

当你的答案和标准答案绝对误差在 0.03 以内时被认为正确。

输入输出样例

输入样例 #1

输出样例 #1

输入样例 #2

输出样例 #2

说明

样例1解释:

第2个课程的前驱课程为第1个课程，所以只有1种合法顺序：1,2，字符串为 ky ，所以一定会出现 ky 。

数据范围:

全部的输入数据满足：1<=n<=100,1<=m<=5。

对于10%的数据，n<=10；

对于另30%的数据，n<=18，m=1。

思路

每个课程最多可能有一个前驱，所以加上一个 $0$ 节点，就变成了一棵有 $n+1$ 个节点的树。

求的是以不同方式遍历一棵树的时候，没有出现（或者出现了）指定的遍历顺序的遍历个数。
