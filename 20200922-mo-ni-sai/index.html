区间(range)

题目描述

有一个$n$个数的序列，一开始所有的数都是$0$，每次可以将一个区间$ [l, r] (l ≤r) $内的数 $+1$，求到达最终状态的最少操作次数。

输入输出格式

输入格式

第一行包含一个正整数$n$，表示序列的长度。

第二行包含$n$个非负整数$a_1, a_2, …, a_n$表示最终的状态。

输出格式

输出的第一行是一个正整数$m$，表示最少的操作次数。

接下来$m$行每行两个正整数 $l_i , r_i$，表示一次操作。你需要保证$1 \le l_i \le r_i \le n$。

保证最少次数 $m \le 10^5$，输出可以以任意顺序输出。

输入输出样例

输入样例 #1

输出样例 #1

说明

数据范围

10%的数据，$n \le 4, m \le 10$

30%的数据，$n \le 10^5, m \le 20$

60%的数据，$n \le 2000, m \le 2000$

100%的数据，$ n \le 10^5, m \le 10^5$

题解

以前做过类似的题目。包括洛谷 9 月月赛的 T3 的十分部分分，就是这题的基础。

变换一下题意，把从 $0$ 加到 $a_i$ 变成从 $a_i$ 减到 $0$，答案都是一样的。

一个简单的结论：设题中所求方案数为 $w$，每当加入一个 $a_i$，其对 $w$ 的贡献是 $\max\{a_i-a_{i-1},0\}$。

至于为什么，一个简单的证明：如果 $a_i$ 小于等于 $a_{i-1}$，那么将对 $a_{i-1}$ 的操作扩展，则 $a_i$ 可以被波及 $a_i$ 次，就减到了 $0$，对答案没贡献。否则 $a_i$ 只能被减到 $a_i-a_{i-1}$，还需要这么多次操作才能变成 $0$。

方案数有了，那么方案呢？

可以发现，每一次对 $w$ 产生贡献，都必定会增加 $a_i-a_{i-1}$ 个区间，而这些区间的左端点都是 $i$。

那么右端点呢？

若 $a_i\le a_{i-1}$，那么其中对 $a_{i-1}$ 的操作扩展了 $a_i$ 次，剩下的没法再进行传递，所以就增加了 $a_{i-1}-a_i$ 个右端点。

因为这些右端点是从前面扩展过来的，所以直接靠左一个一个摆就好了。

到最后还要加上 $a_n$ 个右端点，因为不用再扩展了。

如果用的是不同的方法，可以写一个 spj，来判断自己的答案是不是正确。

一个简单思路就是：

看方案数是不是和标答相同，如果不同输出WA! m diffs!。

看自己的这些区间能不能还原成原序列，用差分实现。如果不能输出"WA! Unsuccessfully Build!"。

都符合就AC!了。

当然呵，这题我是咋做的呢？

昂神上周六洛谷月赛的时候讲了那个结论，我记住了。试了一下发现是对的。

然后琢磨怎么求出方案。

于是想到了大样例。

大样例给的实在是太标准了。

取前几个看一下吧

然后我一看左右端点都是单调递增觉得不太对劲，就找规律。

左端点有五个 $1$，就是 $5-0$ 个，又有一个 $2$，就是 $6-5$ 个……

于是猜到了差分构造的结论。

我又一看右端点，输出了一下 $1009$ 附近的数，2595 2596 2595 2601，果然在第 $1010$ 个数上出现了第一个差分为负的数，而且差正好是 $2596-2595=1$，也就是 $1009$ 的个数。

完全明白了。

两分钟写完。和大样例完全相同（毕竟是看着人家构造的）。封题。

说实话这大样例应该打乱一下顺序。不过出题人可能没想到会有人看着样例想出解法吧hhhhhhhh。

计算(count)

题目描述

给定 $n$，求合法的 $(x_1,x_2,x_3,…,x_{2m})$ 组数。

一组 $x$ 是合法的，当且仅当 $\forall i\in [ 1,2m ] ,x_i \in \mathbb{Z}^+,x_i \mid n,\displaystyle\prod _{i=1}^{2m}x_i\le n^m$

合法的 $(x_1,x_2,x_3,\dots,x_{2m})$ 可能很多，请你输出方案数 $\bmod998244353$。

输入输出格式

输入格式

一行由空格隔开的两个整数，分别是 $n$ 和 $m$。

输出格式

一行表示答案。

输入输出样例

输入样例 #1

输出样例 #1

输入样例 #2

输出样例 #2

说明

【样例解释】

第一个样例中，合法的方案有 (1, 1),(1, 2),(1, 3),(1, 6),(2, 1),(2, 2),(2, 3),(3, 1),(3, 2),(6, 1) 共 10 种。

【数据范围】

$17 \%$的数据满足​，$n\le 50, m=2$

$45 \%$的数据满足，$n\le 100,m\le 3$

$100 \%$的数据满足，$n\le 10^9, m\le 100$

题解

不难发现，$\displaystyle\prod_{i=1}^{2m}x_i\lt n^m$ 的方案数和 $\displaystyle\prod_{i=1}^{2m}x_i\gt n^m$ 的方案数是一样多的。

excuse me？

令 $F(x)=\displaystyle\prod_{i=1}^{2m}x_i$。

如果 $F(x)\lt n^m$，令 $\displaystyle x^\prime=\left(\frac n{x_1},\frac n{x_2},\dots,\frac n{x_{2m}}\right)$，$\displaystyle F(x^\prime)=\frac{n^{2m}}{F(x)}\gt n^m$。

$F(x)\lt n^m$ 和 $F(x^\prime)\gt n^m$ 必定是一一对应的，设前者数量 $s_1$，后者数量 $s_3$，则 $s_1=s_3$。

再设 $F(x)=n^m$ 的数量 $s_2$，那么 $s_1+s_2+s_3=d(n)^{2m}$，就是 $d(n)$ 个数任取 $2m$ 个，取后放回的方案数（$d(n)$ 是约数个数）。

所以答案要求的 $s_1+s_2$ 就是 $\displaystyle \frac{d(n)^{2m}+s_2}{2}$。

所以现在的目标就是求出 $s_2$，也就是 $F(x)=n^m$ 的数量。

设 $n$ 的质因子 $p$ 的指数是 $w$，那么在 $n^m$ 中，$p$ 的指数就是 $w\cdot m$。而且每个 $p$ 都是独立的。

所以就有了转化后要求的内容：$2m$ 个非负整数和为 $sum$ 的方案数。

考虑一个 DP，设 $dp_{i,j}$ 为前 $i$ 个数和为 $j$ 的方案数，初始化 $dp_{0,j}=1$。

因为要求 $x_i\mid n$，所以 $x_i$ 中 $p$ 的指数不能超过 $w$。

所以有转移：$\displaystyle dp_{i,j}=\sum^w_{k=0}dp_{i-1,j-k}$。

发现 $dp$ 数组可以去掉（或者滚掉）一维，然后这个式子可以用一个前缀和优化，把一个位置的转移变成 $\mathcal{O}(1)$。

滚动数组的方式不讲了，说一下怎么去掉一维。

首先自己求自己的前缀和，这时 $[0,w]$ 这个区间相当于已经完成转移了，然后再考虑去掉转移多了的。

$[w+1,w\cdot m]$ 区间内，相应地要减掉（设下标为 $j$）$dp_{j-w-1}$，这样就去掉了多出的转移。

注意这个区间要倒着枚举，不然有后效性。

Comment：滚动数组多香啊，空间虽然大了点（真的是“点”）但少了很多思考，也就少了很多错的可能。

不过平常做题还是要多想想，万一考试就卡这二倍呢？（虽然不太可能）

但考试的时候最好还是拿稳一点，能少丢分才是最主要的。

Evaluation：很有思维性的一道计数题。
