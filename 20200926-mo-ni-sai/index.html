简单的序列（bracket）

题目描述

给定一个括号序列$ s $，满足$ |s| = m$。你需要统计括号序列对$ (p, q) $的数量。

其中$ (p, q)$ 满足$ |p| + |s| + |q| = n $，且$ p + s + q $是一个合法的括号序列。

输入输出格式

输入格式

第一行两个正整数$ n, m。$ 第二行一个长度为$ m $的括号序列，表示$ s $。

输出格式

输出一行一个整数，表示符合条件的$ (p, q) $的数量对$ 10^9 + 7 $取模的值。

说明

样例解释

样例1：

四种方案：

(1)p=空，q=())

(2)p=空，q=)()

(3)p=(,q=))

(4)p=(), q=)

样例2：p=空，q=空

样例3：无方案

【数据范围】

对于 10% 的数据，n ≤ 20；

对于 25% 的数据，n ≤ 200；

对于另外 5% 的数据，n = m；

对于 55% 的数据，n − m ≤ 200；

对于 100% 的数据，1 ≤ m ≤ n ≤ 10^5, n − m ≤ 2000。

题解

复杂度肯定和 $n-m$ 挂钩，不然为啥给 $n-m$ 的范围。

先记录一下 $s$ 内部左右扩号消完（匹配就消）剩的扩号（形如 )))((( 因为只要形状不一样就会被消掉）里的左括号和右括号数量，分别记为 $cntL$ 和 $cntR$。

考虑一个 $\mathcal{O}\left((n-m)^2\right)$ 的 DP，$f_{i,j,0}$ 代表前 $i$ 个括号里面左括号减右括号为 $j$ 个的方案数，这里就钦定左括号数量不小于右括号数量（然后后文枚举的时候就枚举 $p$ 的状态，也就是 $s$ 左边的状态，必须要让 $p$ 消完以后的左括号数量不小于 $cntR$ 才能符合条件，不然枚举出负数了谁负责）。

如果求出这个的话，$p$ 的状态确定了，那么对应的 $q$ 的状态也确定了（因为从左边 $i,j$ 就可以推出右边的 $i,j$）。

然后枚举 $p$ 的状态，左右一乘，就行了。

初始化 $f_{0,0,0}=1$。

然后对于 $i\gt 0$ 的 $f_{i,0,0}$，由于钦定了左括号不小于右括号，所以只能让 $i$ 这里是右括号，也就是 $f_{i,0,0}=f_{i-1,1,0}$。

其他的话考虑第 $i$ 个可能是什么括号，如果是右括号那就从 $f_{i-1,j+1,0}$ 转移，如果是左括号那就从 $f_{i-1,j-1,0}$ 转移，$f_{i,j,0}=f_{i-1,j+1,0}+f_{i-1,j-1,0}$。

可能有细心的同学发现了，我 DP 状态是三维，但是全程只用到了两维。

于是同理可证，设 $f_{i,j,1}$ 为前 $i$ 个括号里面右括号减左括号为 $j$ 个的方案数，则有转移：

$\left\{\begin{array}{l}f_{0,0,1}=1，\\ f_{i,0,1}=f_{i-1,1,1},\\ f_{i,j,1}=f_{i-1,j+1,1}+f_{i-1,j-1,1}\end{array}\right.$。

完全一样不是吗？这是为什么呢？

因为 任何一个左括号多于右括号的序列 倒过来就是 右括号多于左括号的序列。于是。。。状态数完全一样。于是就能把第三维去掉直接用两维数组。

然后枚举 $p$ 的状态 $i\in[0,n-m],j\in[0,i]$，也就是 $p$ 最少有 $0$ 个，最多有 $n-m$ 个扩号，这里面左括号减右括号最多是 $i$（也就是全是左括号）。

如果 $p$ 有 $i$ 个扩号，那么 $q$ 就有 $n-m-i$ 个扩号；如果 $p$ 的左括号减右括号数为 $j$，总的是 $0$，还有 $s$ 的左括号减右括号是 $cntL-cntR$，于是 $q$ 的左括号减右括号数就是 $0-j-cntL+cntR$，但是由于 $q$ 这序列要承接 $s$ 多出来的左括号，所以 $0-j-cntL+cntR$ 这个数一定小于等于 $0$，也就是 $j+cntL-cntR$ 就是 $q$ 的右括号减左括号数。于是乎，$p$ 的 $i,j$ 状态所造成的贡献就是 $f_{i,j,0}\times f_{n-m-i,j+cntL-cntR,1}$（为了便于理解还是把第三维加上）。

于是需要注意特判一下不合法的情况，对应 $j+cntL-cntR$ 这个下标要大于等于 $0$，还要保证 $p$ 多出来的左括号能匹配上 $s$ 多出来的右括号，即 $j\ge cntR$，其实这个解决办法就是直接让 $j$ 从 $cntR$ 枚举就行。

简单的期望（exp）

题目描述

给定一个变量$ x $的初始值。

依次执行$ n $次操作，每次操作有$ p \% $的概率令$ x = x × 2，(100 − p)\% $的概率令$ x = x + 1 $。

假设最后得到的值为$ w $，令$ d $为$ w $的质因数分解中$ 2 $的次数，求$ d $的期望。

输入输出格式

输入格式

第一行三个整数$ x, n, p $，含义见题目描述。

输出格式

一行一个实数，表示$ d $的期望。

如果你的答案与标准答案的误差不超过$ 10^{-6} $，则判定为正确。

输入输出样例

输入样例 #1

输出样例 #1

输入样例 #2

输出样例 #2

输入样例 #3

输出样例 #3

说明

【数据范围】

对于 $20\%$ 的数据，$n\le 20$；

对于 $30\%$ 的数据，$n\le 50$；

对于 $50\%$ 的数据，$n\le 100$；

对于 $100\%$ 的数据，$x\le 10^9,n\le 200,0\le p\le 100$。

题解

首先，质因数分解中 $2$ 的个数即为二进制表示下末尾 $0$ 的个数。

考虑 DP，设 $f_{i,s,j,0/1}$ 表示进行第 $i$ 次操作后，当前数的二进制表示后 $8$ 位为 $s$，第九位为 $0/1$，从第九位开始往前连续 $j$ 位相同的概率。

先来解释后 $8$ 位的原因，因为至少要 $2^8$ 次 $+1$ 操作才可以向 $9$ 进位，而操作数只有 $200$，所以不会对答案造成贡献。

再来解释 $j$ 的用途，假如后 $8$ 位都是 $1$ 第九位也是 $1$，那么我们在进行 $+1$ 的操作后会变成 $1000\ldots$，而 $j$ 是为了计算出这个 $1$ 的位置。

初始化的话，要把关于 $x$ 的初始状态设为 $1$。后八位肯定是 $x\&255$，$i$ 当然就是 $0$，再计算一下第九位往前连续相同的位数就行了。

转移一共有 $8$ 个，每个转移状态加上被转移状态乘对应的概率。

  • 首先看看 $s\times2$ 都能转移到什么状态。

    如果第八位是 $1$，第九位是 $1$，那么可以转移到第九位是 $1$、$j+1$ 的情况，反之第九位是 $0$ 的 $j$ 就得是 $1$。

    如果第八位是 $0$，第九位是 $0$，那么可以转移到第九位是 $0$、$j+1$ 的情况，反之第九位是 $1$ 的 $j$ 就得是 $1$。

  • 再看看 $s+1$ 都能转移到什么状态。

    如果 $s=255$，那么加一会进位，如果第九位是 $0$ 的话，那么会转移到第九位是 $1$，$j=1$ 的情况，否则会转移到第九位是 $0$、$j$ 的情况（因为前面的 $1$ 全都要进位）。

    如果 $s\neq255$，那么加一对第九位和 $j$ 是没有影响的。

答案的话就是把所有 $f_{n,0,j,0/1}$ 都遍历一遍，如果第九位为 $0$，就加上这个概率乘上 $8+j$，否则加上这个概率乘上 $8$。

然后再遍历 $f_{n,s,j,0/1}$，这时候 $j$ 和第九位就无所谓了，把这个概率乘上 $s$ 末尾零的个数就行。

时间复杂度：$\mathcal{O}(2^8n^2)$。

还有一些小细节：如果 $x$ 都没有第九位，那么要设置成 $0$，$j=1$。

简单的操作（merge）

题目描述

有个包含$ n $个点，$ m $条边，无自环和重边的无向图。

对于两个没有直接连边的点$ u,v $，你可以将它们合并。具体来说，你可以删除$ u,v $及所有以它们作为端点的边，然后加入一个新点$ x $，将它与所有在原图中与$ u $或$ v $直接连边的点连边。

你需要判断是否能通过若干次合并操作使得原图成为一条链，如果能，你还需要求出这条链的最大长度。

输入输出格式

输入格式

第一行两个正整数$ n,m $，表示图的点数和边数。

接下来$ m $行，每行两个正整数$ u,v $，表示$ u $和$ v $之间有一条无向边

输出格式

如果能通过若干次合并操作使得原图成为一条链，输出链的最大长度，否则输出$ -1 $

说明

样例1解释：

只要合并4和5就变成一条长度为3的链。1-2-3-x(4,5)

【数据范围】

对于$30\%$的数据，$n\lt10$

对于$70\%$的数据，$n\lt2,000$

对于$100\%$的数据，$n\leq 1,000,m\leq 10^5$

题解

发现有三元环是不可能成链的，因为想要合并必须要不相邻。推广下，就是有奇环都不行，因为通过每次减少两个点总会到达三元环的状态。

嗯，所以这图有解当且仅当没奇环，这不就是二分图吗？

先染个色，有冲突就无解。

然后考虑怎么求方案数。

如果把所有连通块都缩成链了，那么我们可以通过合并两个链的任意端点把两个链合成一个链，长度为原先两个链长度的和。

于是我们就可以求出所有连通块的链长，然后加起来。

那考虑一个连通块内怎么做呢？

合并后肯定会有一个点 $s$ 的度数为 $1$，那么到 $s$ 所有距离为 $1$ 的点只能有一个，也就是需要把所有到 $s$ 距离为 $1$ 的点合并到一起。同理，需要把所有到 $s$ 距离相同的点合在一起。

你可能会问了，为啥不能把距离不同的点合在一起？

举一个简单的例子：

图一中若合并 $x$ 与 $y$，距离分别为 $2$ 和 $1$，合并完会变成图二的样子，发现图中出现了一个三元环，是不符合规定的。

你可能又会问了：我合并染成同色（距离同奇或同偶）的点不就不会出锅了吗？

如图三中若合并 $x$ 与 $y$，距离分别为 $1$ 和 $3$，合并完会变成图四的样子。这次虽然保证了有解，但是从根开始能合并出的最长链短了啊（原来是 $3$，现在只剩 $2$ 了），肯定不是最优的解法。

现在来稍微理性一点证明：为保证有解，一定是要合并距离奇偶性相同的点。若合并距离不同的点，不妨设为 $x$ 和 $y$，距离分别为 $dis_x,dis_y(dis_x<dis_y)$，则 $dis_y-dis_x\ge 2$，但如果合并了 $x$ 与 $y$，由于要把所有边连在新点上，所以根到新点的距离是 $dis_x$，而 $y$ 由于和新点连了边，根到它的距离变成了 $dis_x+1$，可是 $dis_x+1\lt dis_y$，这样从根开始连的最大链长可能减小，无法使答案更优。所以只能合并距离相同的点。

既然只能合并距离相同的点，那么从一个点开始的最大链长就是它到其他 连通块内的点 的最长距离。然后我们还需要最大化这个东西，就求出连通块内每个点到其他点的最短路的最大值，作为连通块的答案累计进总的答案。

由于没有边权，可以直接 $\mathcal{O}(n)$ bfs，每个点都做一遍，总时间复杂度 $\mathcal{O}(n^2)$，可以通过本题。
