牛牛的方程式

时间限制：C/C++ 1秒，其他语言2秒

空间限制：C/C++ 262144K，其他语言524288K

64bit IO Format: %lld

题目描述

牛牛最近对三元一次方程非常感兴趣。众所周知，三元一次方程至少需要三个方程组成一个方程组，才有可能得出一组解。

牛牛现在想要知道对于方程 $ax+by+cz=d$ 中有没有至少存在一组 $\{x,y,z\}$ 的解，且 $x,y,z$ 都为整数，使得方程式成立。

输入描述:

第一行输入一个正整数 $T$，表示测试点中测试样例的组数。

接下来 $T$ 行，每行四个整数 $a,b,c,d$ 表示方程中的 $a,b, c,d$。

输出描述:

如果至少存在一组 $\{x,y,z\}$ 能够满足方程式等式成立，且 $x,y,z$ 均为整数，请输出”YES”，否则请输出”NO”。

示例1

输入

输出

说明

$1*3+(-1)*1+(-1)*2=0$
得到一组 $x,y,z$ 的解为 $\{1,-1,-1\}$ 为整数使得等式成立，所以输出”YES”。

不存在 $x,y,z$ 为整数使得方程 $2x+8y+8z=3$ 成立，所以输出”NO”。

备注:

对于 $10\%$ 的测试数据，保证
$T=1,-10 \leq a,b,c,d \leq10$；

对于 $30\%$ 的测试数据，保证 $-100 \leq a,b,c,d\leq 100$；

对于 $100\%$ 的测试数据，保证 $-10^{18}\leq a,b,c,d\leq 10^{18} ,1 \leq T \leq 100$。

题解

大意：判断三元一次方程有无整数解。

类似于二元一次方程 $ax+by=c$ 有整数解条件 $\gcd(a ,b)\mid c$，三元一次方程 $ax+by+cz=d$ 有整数解的条件为 $\gcd(a,b,c)\mid d$。

证明：

由二元一次方程有整数解条件可知 $ax+by$ 可以表示 $\gcd(a,b)$ 的任意整数倍，所以三元一次方程 $ax+by+cz=d$ 就可以转化为 $\gcd(a,b)\cdot x+cz=d$，而其有整数解条件为 $\gcd(\gcd(a,b),c)\mid d$，即 $\gcd(a,b,c)\mid d$。

Q.E.D.

代码实现时要注意 $a,b,c$ 均为 $0$ 的情况，不然对 $0$ 取模会出错。

牛牛的猜球游戏

时间限制：C/C++ 1秒，其他语言2秒

空间限制：C/C++ 262144K，其他语言524288K

64bit IO Format: %lld

题目描述

牛牛和牛妹在玩猜球游戏，牛牛首先准备了 $10$ 个小球，小球的编号从 $0\sim9$。首先牛牛把这 $10$ 个球按照从左到右编号为 $0,1,2,3,\dots,9$ 的顺序摆在了桌子上，接下来牛牛把这 $10 $ 个球用 $10$ 个不透明的杯子倒扣住。
牛牛接下来会按照一定的操作顺序以极快的速度交换这些杯子。
换完以后他问牛妹你看清楚从左到右的杯子中小球的编号了么？

由于牛妹的动态视力不是很好，所以她跑来向你求助。你在调查后发现牛牛置换杯子其实是有一定原则的。

具体来讲，牛牛有一个长度大小为 $n$ 的操作序列。

操作序列的每一行表示一次操作都有两个非负整数 $a,b$，表示本次操作将会交换从左往右数第 $a$ 个杯子和从左往右数第 $b$ 个杯子（$a$ 和 $b$ 均从 $0$ 开始数）。请注意是换杯子，而不是直接交换 $a$ 号球和 $b$ 号球。

牛牛和牛妹一共玩了 $m$ 次猜球游戏，在每一轮游戏开始时，他都将杯子中的小球重置到从左往右依次为 $0,1,2,3,\dots,9$ 的状态。

然后在第 $i$ 轮游戏中牛牛会按照操作序列中的第 $l_i$ 个操作开始做，一直做到第 $r_i$ 个操作结束（$l$ 和 $r$ 的编号从 $1$ 开始计算）。

由于你提前搞到了牛牛的操作序列以及每一次游戏的 $l,r$。请你帮助牛妹回答出牛牛每一轮游戏结束时，从左至右的杯子中小球的编号各是多少。

输入描述:

首先输入一行两个正整数 $n,m$，表示操作序列的长度以及进行游戏的次数。

接下来 $n$ 行每行两个非负整数 $a,b$，表示交换左数第 $a$ 个杯子和左数第 $b$ 个杯子（$a,b$ 均从 $0$ 开始数起）。

接下来 $m$ 行每行两个正整数 $l,r$ 表示该轮游戏中牛牛从第 $l$ 个操作开始做，一直做到第 $r$ 个操作结束（$l$ 和 $r$ 的编号从 $1$ 开始计算）。

输出描述:

对于每一轮游戏，输出一行 $10$ 个非负整数，表示从左至右每一个杯子中小球，输出的整数之间用空格隔开，行末不允许有多余空格。

$1 \leq n,m \leq 500$；

对于 $60\%$ 的测试数据，保证 $1 \leq n,m \leq 4 \times 10^4$；

对于 $60\%$ 以外另 $10\%$ 的数据，保证输入的 $a,b \in \{0,1,2\}$；

对于 $100\%$ 的测试数据，保证 $1 \leq n,m \leq 10^5,0 \leq a,b \leq 9 ,1 \leq l\leq r \leq n$。

牛牛的凑数游戏

时间限制：C/C++ 1秒，其他语言2秒

空间限制：C/C++ 262144K，其他语言524288K

64bit IO Format: %lld

题目描述

对于一个多重数集 $S$，对于一非负整数 $x$，若存在 $S^\prime\subseteq S$ 且 $S^\prime$ 中所有数字之和恰好等于 $x$，则说 $S$ 可以表示 $x$。

显然对于任意的多重数集都可以表示 $0$，因为空集是所有集合的子集。

牛牛定义集合 $S$ 的最小不能表示数为，一个最小的非负整数 $x$，$S$ 不能表示 $x$。举个例子来说，例如 $S=\{1,2,3,8,9\}$，那么集合 $S$ 的最小不能表示数就为 $7$。

因为子集 $\{\}$ 和为 $0$，子集 $\{1\}$ 的和为 $1$，子集 $\{2\}$ 的和为 $2$，子集 $\{1,2\}$ 的和为 $3$，子集 $\{1,3\}$ 的和为 $4$，子集 $\{2,3\}$ 的和为 $5$，子集 $\{1,2,3\}$ 的和为 $6$。但是无法找到子集权值和恰好为 $7$ 的子集，所以 $7$ 无法表示。

现在有一个长度大小为 $n$ 的正整数数组，牛牛每次选择一个区间 $[l,r]$，他想要知道假定给出的多重数集为 $\{a_l,a_{l+1},\dots,a_{r}\}$ 时，该集合的最小不能表示数是多少。

输入描述:

第一行输入两个正整数 $n,m$。

接下来一行输入 $n$ 个正整数表示输入的正整数数组。

接下来 $m$ 行，每行输入两个正整数 $l,r$ 表示查询的区间。

输出描述:

对于每一个查询，输出最小不能表示数。

示例1

输入

输出

备注:

对于前 $10\% $ 的测试数据，保证
$1 \leq n,m \leq 10 ,1 \leq a_i \leq 100$；

对于前 $20\%$ 的测试数据，保证 $1 \leq n,m \leq 500$；

对于另 $10\%$ 的测试数据，保证输入的单调非降；

对于另 $10\%$ 的测试数据，保证输入的为 $2$ 的非负整数幂；

对于 $100\%$ 的测试数据，保证 $1 \leq n,m \leq 10^5,1 \leq a_i \leq 10^9 ,1 \leq l \leq r \leq n$。

题解

对区间内的数排序，可以发现若 $sum_i+1<a_{i+1}$，则最小不能表示出来的就是 $sum_i+1$。
