题目描述

你有一个 $N$ 个点，$M$ 条边的带权无向连通图。

现在你需要回答 $Q$ 个询问，每个询问给出两个点，问这两个点之间的最短路是多长。

输入格式

第一行包含三个正整数 $N,M,Q$。

接下来 $M$ 行每行三个正整数 $u,v,c$，表示 $u$ 和 $v$ 之间有一条长度为 $c$ 的无向边。

接下来 $Q$ 行每行两个正整数 $u$ 和 $v$。

输出格式

$Q$ 行，每行对应一个询问的答案。

数据规模与约定

对于 $30\%$ 的数据，$N \leq 300$。

对于另外 $20\%$ 的数据，$M=N-1$。

对于 $100\%$ 的数据，$N \leq 100000,M \leq N+20,Q \leq 100000,c \leq 10^9$，给出的边中无重边、自环。

时间限制：$3 \text{s}$。

空间限制：$256 \text{MB}$。

题解

把这篇文发出来主要是为了这道题，原题CF1051F The Shortest Statement。

之前 sky 讲过，于是这场比赛成了四道原题。

好（蒿）！

发现如果不看数据范围的话这就是一道裸的全源最短路，用 Floyd 算法 $\mathcal{O}(n^3)$ 可以解决这道题。

但是发现 $n\le10^5$，当然可以信仰跑 Floyd，不过过不了。

但是又发现 $m\le n+20$，这不就是树加上几条非树边嘛。

对于树上的全源最短路径（雾），很好求。如果这题是一棵树就好了，于是我们可以把这张图的一棵生成树求出来，而非树边可能使得生成树上的路径并非最短路径，于是对每一条非树边的两个端点求一遍单源最短路。用 Dijkstra 算法，最多有 $m-n+1=21$ 条非树边，也就最多跑 $42$ 遍单源最短路，是能过去的。

这样在查询两个点最短路的时候，生成树上路径长度可以被计入答案，每条非树边两个端点其中之一到两个查询点的最短路和也可以被计入答案，查询最小值即可。

很思维的一道题。

注：不一定非要是最小生成树，因为最小生成树不一定最优，那么也就没必要让总的边权值最小。
