[TJOI2015]弦论

题目描述

为了提高智商，ZJY 开始学习弦论。这一天，她在《String theory》中看到了这样一道问题：对于一个给定的长度为 $n$ 的字符串，求出它的第 $k$ 小子串是什么。你能帮帮她吗?

输入输出格式

输入格式

第一行是一个仅由小写英文字母构成的字符串 $s$。

第二行为两个整数 $t$ 和 $k$，$t$ 为 $0$ 则表示不同位置的相同子串算作一个，$t$ 为 $1$ 则表示不同位置的相同子串算作多个。$k$ 的意义见题目描述。

输出格式

输出数据仅有一行，该行有一个字符串，为第 $k$ 小的子串。若子串数目不足 $k$ 个，则输出 $-1$。

说明

数据范围

对于 $10\%$ 的数据，$n\leq 1000$。

对于 $50\%$ 的数据，$t = 0$。

对于 $100\%$ 的数据，$1\leq n \leq 5 \times 10^5$，$0\leq t \leq 1$，$1\leq k \leq 10^9$。

SAM 做法

建出后缀自动机，然后在自动机上维护一下每个点处的 endpos 集合大小（也就是这个子串的出现次数），在代码里是 $sz$。

维护方法：要在自动机上把 $fa$ 的 $sz$ 加上当前节点的 $sz$，也就是在 parent 树上 DP 一遍。

只有这个也不太够，要像平衡树一样按字典序求出第 $k$ 小还需要求一个 $g$ 数组，也就是经过当前点的子串数量。

维护方法：这个可以逆着拓扑序 DP 一遍。

至于具体写什么代码，dfs 应该是最好理解的，毕竟这两个要维护的东西本质上都是在 DAG（树也是 DAG 吧？）做 DP。

由于 SAM 存储的东西（字符串）的特殊性，可以采用按长度排序的方式求出其拓扑序，因为字符串被访问时肯定是长度递增的，甚至因为排序的关键字是长度可以用 $\mathcal{O}(n)$ 桶排，于是逆拓扑序 bfs 就很方便了。

代码虽然不太好理解但写起来也挺方便的。
