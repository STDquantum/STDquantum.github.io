# P4124 [CQOI2016]手机号码

## 题目描述

人们选择手机号码时都希望号码好记、吉利。比如号码中含有几位相邻的相同数字、不含谐音不吉利的数字等。手机运营商在发行新号码时也会考虑这些因素，从号段中选取含有某些特征的号码单独出售。为了便于前期规划，运营商希望开发一个工具来自动统计号段中满足特征的号码数量。 工具需要检测的号码特征有两个：号码中要出现至少 $3$ 个相邻的相同数字；号码中不能同时出现 $8$ 和 $4$。号码必须同时包含两个特征才满足条件。满足条件的号码例如：13000988721、23333333333、14444101000。而不满足条件的号码例如：1015400080、10010012022。 手机号码一定是 $11$ 位数，前不含前导的 $0$。工具接收两个数 $L$ 和 $R$，自动统计出 $[L,R]$ 区间内所有满足条件的号码数量。$L$ 和 $R$ 也是 $11$ 位的手机号码。

## 输入输出格式

### 输入格式

输入文件内容只有一行，为空格分隔的 $2$ 个正整数 $L,R$。

### 输出格式

输出文件内容只有一行，为 $1$ 个整数，表示满足条件的手机号数量。

## 输入输出样例

### 输入样例 #1

```
12121284000 12121285550
```

### 输出样例 #1

```
5
```

## 说明

样例解释：满足条件的号码： 12121285000、 12121285111、 12121285222、 12121285333、 12121285550。 数据范围：$10^{10}\leq L\leq R<10^{11}$。

我相信你没明白```solve1```是啥意思，这不是```solve2```要研究的，所以里面的参数暂且用blah, blah, blah代替。

然后就进入了最关键的部分，DP数组的设计与求解。

梳理一下已有的限制条件：

- $4$ 和 $8$ 不能都有
- 至少$3$位连号

我们就可以用$f3$表示是否形成了三位连号，$f4$代表有没有$4$，$f8$代表有没有$8$。这几个的取值范围均为$[0,1]$。

光是这样还不够，因为如果是三位连号的话我们需要知道这位的前两位数字是什么，于是我们用$pre$代表前一位的数字，$pree$代表前一位的前一位数字。这两个的取值范围均为$[0,9]$。

然后还有一个小小的限制条件，就是上一位是不是小于它能取到的最大数字，把它设为$xn$。比如$1946$中现在在研究个位，若$pre<4$即$xn=1$，也就是$191X-193X$的话，$X$的取值范围是$[0,9]$；而如果$pre=4$即$xn=0$，也就是$194X$的话，则$X$只能在$[0,6]$范围内。$xn$取值范围$[0,1]$。

再加上一个$now$表示第几位。取值范围是$[0,10]$，为啥不取$11$呢，一会在代码里看

所以就可以设出一个冗长的DP状态
$$
f[now][pre][pree][f3][xn][f4][f8]
$$
而```solve1```函数也应运而生

```cpp
ll solve1(ll now, ll pre, ll pree, bl f3, bl xn, bl f4, bl f8) {
	if (f4 & f8) return 0; // 如果4和8全有不符合规范
	if (now <= 0) return f3;
    /*
    很巧妙的减少码量措施。
    已经递归完毕了，如果有三个连号说明找到了一个可行的号码（因为4和8的问题在前面排除了）
    如果有三个连号的话，f3=1，对应找到一个；如果没有，f3=0，正好也满足题意。
    所以return f3完美解决问题。
    */
	if (~f[now][pre][pree][f3][xn][f4][f8]) {
        // 取反的话除了-1其他数都为真，对应着已经更新过这个DP状态。因为初始化是-1。
		return f[now][pre][pree][f3][xn][f4][f8];
	}
	ll tmp = 0, lim = !xn ? num[now] : 9; 
    /*
    lim就是前文提到的xn作用中很重要的一环。
    如果xn=1，那么这一位取什么都行，若xn=0，只能取到这一位的最大数字，即上文例中的6。
    */
	for(ll i = 0; i <= lim; ++i) {
		tmp += solve1(now - 1, i, pre, f3 | (i == pree && i == pre), xn | (i < lim), f4 | (i == 4), f8 | (i == 8));
        /* 
        now：now-1就是往小里递归求解
        pre&pree：之前的前两位为pree,pre，之后的变成了pre,i
        f3：如果当前位取的i与之前两位都相等，那么说明出现了三位连号
        xn：如果i小于最大能取到的数字，后面随便取
        f4&f8：更新i是否等于4或8
        */
	}
	return f[now][pre][pree][f3][xn][f4][f8] = tmp; // 赋值+返回
}
```

现在可以解决```solve2```函数里递归的起点了。

因为之前一位的信息会传递到函数里，所以直接从第$10$位开始就行，$pre$ 是最高位数字，$pree$就是$0$。剩下的不用我多说了吧？

```cpp
solve1(10, i, 0, 0, i < num[len], i == 4, i == 8);
```

完整代码：

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace STDquantum {

typedef long long ll;
typedef bool bl;
ll r, l, f[12][10][10][2][2][2][2], num[12];
ll solve1(ll now, ll pre, ll pree, bl f3, bl xn, bl f4, bl f8) {
	if (f4 & f8) return 0;
	if (now <= 0) return f3;
	if (~f[now][pre][pree][f3][xn][f4][f8]) {
		return f[now][pre][pree][f3][xn][f4][f8];
	}
	ll tmp = 0, lim = !xn ? num[now] : 9;
	for(ll i = 0; i <= lim; ++i) {
		tmp += solve1(now - 1, i, pre, f3 | (i == pree && i == pre), xn | (i < lim), f4 | (i == 4), f8 | (i == 8));
	}
	return f[now][pre][pree][f3][xn][f4][f8] = tmp;
}
ll solve2(ll x) {
	if (x < 1e10) return 0;
	memset(f, -1, sizeof(f));
	long long len = 0, tmp = 0;
	while (x) {
		num[++len] = x % 10;
		x /= 10;
	}
	for(int i = 1; i <= num[len]; ++i) {
		tmp += solve1(10, i, 0, 0, i < num[len], i == 4, i == 8);
	}
	return tmp;
}
inline void main() {
	scanf("%lld%lld", &l, &r);
	printf("%lld", solve2(r) - solve2(l - 1));
}

} // namespace STDquantum

int main() {
	STDquantum::main();
	return 0;
}
```
