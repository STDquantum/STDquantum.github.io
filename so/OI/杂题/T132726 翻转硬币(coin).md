# AC代码

感谢刘皓铭大佬的代码（洛谷名```lhm_```）

若是有些细节看注释不明白，下文有详细解释。

```cpp
#include <bits/stdc++.h>
using namespace std;
const int N = 5000010, K = 35, inf = 1e9;
int n, k, m, tot, all;
	/*
	n, k, m: 硬币数、最后向上硬币个数、可翻转次数
	tot: 最后操作的边界个数（含左右边界）
	all: 最终状态
	*/
int f[N], d[N], a[N], t[N], dis[K][K], dir[2] = {1, -1};
	/*
	f: 状压DP的数组
	d: 求最短路的距离数组
	a: m次操作分别的区间长度
	t: 最终标记的区间端点
	dis: 最短路，即翻转一个区间的最小值
	dir: 方向，即左或右
	*/
bool tag[N]; // 标记一个点是否在最终要被翻。
template<typename T> inline void read(T &x) { // 快读
	x = 0;
	char ch = getchar();
	bool f = 0;
	while (!isdigit(ch)) {
		if (ch == '-') f = 1;
		ch = getchar();
	}
	while (isdigit(ch)) {
		x = (x << 1) + (x << 3) + (ch ^ 48);
		ch = getchar();
	}
	if (f) x = -x;
}
void bfs(int s) { // 一个用来求最优方案的最短路
	for(int i = 0; i <= n; ++i) d[i] = inf;
	queue<int> q;
	q.push(t[s]), d[t[s]] = 0;
	while (!q.empty()) {
		int x = q.front(), nx;
			/*
			x: 队首，即当前所需要最短路的一个端点（下标）
			nx: 操作后路径的另一个下标
			*/
		q.pop();
		for(int i = 1; i <= m; ++i) { // 枚举m种操作
			for(int j = 0; j <= 1; ++j) { // 枚举对a[i]操作的方向
				nx = x + dir[j] * a[i]; // 向左或右进行区间翻转操作
				if (nx < 0 || nx > n) continue; // 如果越界（不可能这样翻转）
				if (d[nx] > d[x] + 1) d[nx] = d[x] + 1, q.push(nx); // 更新并继续最短路
			}
		}
	}
	for(int i = 1; i <= tot; ++i) {
		if (dis[s][i] > d[t[i]]) {
			dis[s][i] = dis[i][s] = d[t[i]]; // 更新全局最短路
		}
	}
}
int main() {
	read(n), read(k), read(m);
	int x;
	for(int i = 1; i <= k; ++i) {
		read(x), tag[x] = 1; // 读入最终会被翻的硬币位置，打上标记
	}
	for(int i = 1; i <= m; ++i) {
		read(a[i]); // 读入一次能翻转几个
	}
	for(int i = 1; i <= n; ++i) {
		if (tag[i]) { // 如果被打上了标记
			t[++tot] = i - 1;
			while (tag[i]) ++i;
			t[++tot] = i - 1;
			/*
			思想为找到一个连续的标记区间，将左端点设为区间左端点-1，
			右端点设为区间右端点，在操作时候区间左开右闭。
			具体见后文举例解释。
			*/
		}
	}
	all = (1 << tot) - 1; 
		/*
		如tot=4时，all为15，二进制是1111，
		即为对四个边界都进行了操作，即为终态。
		*/
	for(int i = 1; i <= all; ++i) f[i] = inf; // 初始化f数组
	for(int i = 1; i <= tot; ++i) {
		for(int j = i + 1; j <= tot; ++j) {
			dis[i][j] = dis[j][i] = inf; // 初始化距离数组
		}
	}
	for(int i = 1; i <= tot; ++i) bfs(i); // 求最短路
	for(int s = 0; s <= all; ++s) { // 二进制枚举起始状态
		int pos = inf; // 操作点
		for(int i = 1; i <= tot; ++i) { // 枚举区间端点
			if (s & (1 << (i - 1))) continue; 
			/*
			i-1的原因是若s为二进制0100，这个1对应的是第三位，而第三位对应的是1<<2。
			如果s在这里被操作过，自然不用再次操作它了，否则无法保证每个点操作奇数次
			*/
			if (pos == inf) { // 在此起始状态还未操作过
				pos = i; // 标记操作点
				continue;
			}
			int ns = s | (1 << (i - 1)) | (1 << (pos - 1));
			/*
			s按位或后，即为对这两点操作后的状态。
			*/
			f[ns] = min(f[ns], f[s] + dis[i][pos]); // 更新状态
		}
	}
	if (f[all] == inf) { // 若最终状态没有被更新过
		puts("-1"); // 无解
	} else {
		printf("%d\n", f[all]); // 输出答案
	}
	return 0;
}
```

# 主要思路解释

## 转换为区间端点操作

我们可以发现，对于下图要实现的黑色区间翻转，可以转化为在点上进行奇数次操作，因为如图的蓝、红、绿三种区间翻转的操作，最后得到的结果是完全一样的。所以我们可以忽略区间的存在，以及左右端点的区别（见下文）。

{% asset_image 2.png %}

## 区间左开右闭

我们可以发现，对于下图的情况（黑色实心点代表最终需要被翻转的，空心点最终不需要被反转），如若按照如下的方式，将区间端点看做标记点（$t$数组）进行操作，则按红色进行操作时是可以正常翻转的，但是按照蓝色操作时，会有两个点被翻转过头，不能达到想要的效果。

{% asset_image 3.png %}

所以我们需要把左端点向左移一个位置，并在翻转时不操作该位置，也就是操作区间左开右闭，就能解决这个问题。如图，蓝色操作与红色操作左右端点均为$t$数组上的标记点，并且操作左开右闭，可以发现，这样操作的话，两种操作在效果上是完全等价的。（可能会有些抽象，建议手动模拟一下）

{% asset_image 4.png %}

## 最短路

转化为区间端点操作后，可以发现区间翻转操作满足最短路原理，即若翻转$x$次可以达到翻转$y$次的效果（$x<y$），那么翻转$x$次一定更优，我们就可以用最短路来解决两点之间操作的最少次数问题。

如下图，若想翻转红色区间，那蓝色操作一定比绿色操作要优。

（$dis$数组是不管翻转后是否满足条件的，只是讨论翻转所需的最小值。）

{% asset_image 5.png %}

## 状态压缩

看数据范围可以发现$k$是很小的，只有$10$，而$t$数组所标记出的点数最多有$20$（对应每一个区间都是单独的一个点的状态），而$2^{20}=1048576$，操作复杂度在可接受的范围内，所以考虑状态压缩，$0$和$1$分别代表为操作过和已操作过该点。
