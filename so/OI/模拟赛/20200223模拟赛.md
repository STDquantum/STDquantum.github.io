# 交换(swap)

## 限制

时间限制：$2000$MS

内存限制：$524288$KB

## 题目描述

给定正整数$n$和两个排列$A,B$，我们用$$A_i$$和$$B_i$$分别表示排列$A$和排列$B$的第$i$项，其中$1\le i\le n$。在每个排列中，正整数$1,2,\cdots ,n$恰好各出现了一次。

你可以对排列$A$中的元素进行若干次交换，进行交换需要一定的代价。每次交换需指定两个整数$i,j$，满足$1\le i,j \le n$且$i\neq j$。该操作将交换$$A_i$$与$$A_j$$的值，同时交换的代价为$|i-j|$。你的目标是通过不超过$n$次交换使得排列$A$变成排列$B$，要求交换的总代价最小。
## 输入格式
输入的第一行包含一个正整数$n$。

第二行包含$n$个正整数，表示$$A_1,A_2,\cdots ,A_n$$。

第三行包含$n$个正整数，表示$$B_1,B_2,\cdots ,B_n$$。
## 输出格式
第一行输出一个整数$t$，表示交换次数。

接下来$t$行，依次描述每次交换：每行包含两个整数$i,j(1\le i,j\le n\;$且$\;i\neq j)$，表示交换$$A_i$$与$$A_j$$的值。如果有多种使得交换总代价最小的方案，输出任意一种方案即可。
## 输入 样例 1
```
4
4 2 1 3
3 2 4 1
```
## 输出 样例 1
```
2
3 4
1 3
```
## 输入 输出 样例 1 说明
样例中进行了$2$次交换。
第$1$次交换$$A_3$$和$$A_4$$的值，此时排列$A$为$4,3,2,1$。
第$2$次交换$$A_1$$和$$A_3$$的值，此时排列$A$为$3,2,4,1$，与排列$B$相同。该方案的总代价为$|3-4|+|1-3|=3$，为所有方案中最小的。

## 数据规模与约定

对于每个测试点，如果你的输出不合法，或者输出方案的代价不是所有满足条件的方案中最小的，该测试点得$0$分。
如果输出合法且代价是最小的，但交换次数$t$超过了$n$，你可能还能获得部分分：

- 如果$t>2n$，你将得到该测试点$0\%$的分数；
- 如果$n< t \le 2n$，你将得到该测试点$40\%$的分数；
- 如果$0\le t \le n$，你将得到该测试点$100\%$的分数。

所有测试数据的范围和特点如下表所示（“有”表示保证测试点具有相应性质，“无”表示不保证其具有相应性质）：

# 最长重复子串(string)
## 限制

时间限制：2000MS

内存限制：524288KB

## 题目描述
字符串$S$被称为重复子串，当且仅当存在一个字符串$A$，使得$S=AA$。例如 abcabc、xyxyxyxy 是重复子串，而 abcab、xyxyxy 不是重复子串。注意空串也是重复子串。给定字符串$S$，求$S$的一个最长子串$T$，使得$T$是重复子串。你只需输出最长重复子串的长度，即$|T|$的值。

## 输入格式
输入共一行，为一个只包含小写字母的字符串$S$。
## 输出格式
输出一行一个整数，表示答案$|T|$。
## 输入 样例 1
```
ababcabcbc
```
## 输出 样例 1
```
6
```
## 输入 输出 样例 1 说明
样例中， abab、abcabc、bcbc都是$S$的重复子串，最长的为 abcabc ，长度为$6$。
## 输入 样例 2
```
abcba
```
## 输出 样例 2
```
0
```
