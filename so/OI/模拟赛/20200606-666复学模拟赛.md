Barca昨天强势实现三连冠，这势必要进行庆祝，于是庆祝用蛋糕就这样诞生了，这是一块矩形蛋糕，它足足有$N*M$个小蛋糕组成，每个蛋糕的美味指数为$T_{i,j}$。

## 题解

题意简述：

$N*M$的矩阵上每格有一个非负整数。先将矩阵横向切成$A$块，再将每块切成$B$块。

这$A*B$块中，将每块中每格的数加起来。要使和最小的那块尽量大。

 

算法分析：

这道题初看会觉得无从下手，因为没有明显的算法特征，所以分析就变得尤为重要。

首先我想说的是一种不甚科学的通法：如果题目中出现要最大化一个最小值或者最小化一个最大值的时候，往往跟二分有关。这个方法不是百分百正确的，但是确实适用非常多的题目。即使不适用，也提供一个思考的方向。如果能想到这里，这题就变得很容易了。

当我们二分出一个答案$ans$的时候，题目就转换为一个判定性问题，即是否有一种方案使得各格的数加起来不小于$ans$。需要注意，一个求最优的问题通过二分转换为判定性问题也是很常见的手段。

转为判定性问题一定要有利于解决题目的地方。本题有一个关键的条件，就是每格的数都是非负整数。也就是说如果在达到$ans$的情况下，我们应该使切出的方块尽量少，把更多方块留给后面，这样后面达到$ans$的机会也就更大。这个贪心的想法同时适用于切$A-1$刀和切$B-1$刀：切$A-1$刀时，如果当前切出$i$~$j$行已经符合条件，那么就不要再多切$j+1$；切$B-1$刀时，如果当前从$i$列切到$j$列已经符合条件，那么就不要多切$j+1$。

这样算法就很清晰了：首先二分出答案$ans$，进入判定；从$1$开始枚举$A-1$刀中的第一刀应该切在哪里，并假设切下；在假设的情况下从$1$开始枚举$B-1$刀中的第一刀应该切在哪里，必须在刚刚好不小于$ans$的位置切，然后从该位置开始枚举下一刀；如果切出了$B$块，那么$A-1$刀中的第一刀可以在该枚举的位置，然后从该位置开始枚举下一刀；如果切出了$A$块，那么$ans$可行，否则不可行；结束该轮判定继续二分。

# 电路维修

## 题面

时间限制：$1000MS$   内存限制：$128M$   cir.cpp

### 题目描述

Elf 是来自Gliese 星球的少女，由于偶然的原因漂流到了地球上。在她无依无靠的时候，善良的运输队员Mark 和James 收留了她。Elf 很感谢Mark和James，可是一直也没能给他们帮上什么忙。

有一天 Mark 和James 的飞行车没有办法启动了，经过检查发现原来是电路板的故障。飞行车的电路板设计很奇葩，如下图所示：

{% asset_image 1.gif %}

电路板可以表示为一个 $R$ 行 $C$ 列的方格表，每个格子中都有一个标准件，连接一条对角线上的两个接点。标准件的方向可以旋转，从而连接另一条对角线的两个接点。电源与左上角的接点相连，而发动机与右下角的接点相连。Mark 和James 发现现在电源和发动机根本就没有连上。出于对这种奇葩设计的尊重，他们只能通过旋转标准件来维修电路。

Elf 想要帮助他们，让他们旋转标准件的次数尽量小，但是电路的规模实在是太大了，于是她找到了你，希望你能够帮助她解决这个问题。

### 输入格式

输入文件包含多组测试数据。第一行包含一个整数T 表示测试数据的数目。

对于每组测试数据，第一行包含正整数 $R$ 和 $C$ ，表示电路板的行数和列数。

之后 $R$ 行，每行 $C$ 个字符，字符是"/"和"\\"中的一个，表示标准件的方向。

### 输出格式

对于每组测试数据，在单独的一行输出一个正整数，表示所需的最小旋转次数。

如果无论怎样都不能使得电源和发动机之间连通，输出 ```NO SOLUTION```。

### 样例输入 cir.in

```
1
3 5
\\/\\
\\///
/\\\\
```

### 样例输出  cir.out

```
1
```

### 数据规模与约定

样例的输入对应于题目描述中的情况。

只需要按照下面的方式旋转标准件，就可以使得电源和发动机之间连通。

{% asset_image 2.gif %}

数据范围与约定

对于$40\%$ 的数据，$R,C\le 5$。

对于 $100\%$ 的数据，$R,C\le 500$，$T\le 5$。

## 题解

- 关键词：

  最短路

- 题意简述：

  改变最少的路的连接方向，使起点终点相通。

- 算法简述：

  对于每条路，只有改跟不改两种情况，改会贡献$1$的代价，不改不贡献代价，也就是贡献$0$的代价，要求最小代价。这启发我们建图连边，将问题转化为我们所熟知的最短路问题。

  原图中每个节点作为点，对于一个正方形中对角线相连的那两个点之间连一条代价为$0$的边，没有对角线相连的那两个点之间连一条代价为$1$的边，接下来只要做一遍最短路SPFA即可。

  这样做的复杂度大约是边数*常数*测试数据组数，即常数$*500*500*2*5$，大约是几百万到几千万。对于大数据，是容易超时的。这题有个神奇的优化方法，神奇之处是它的SPFA跟我们一般的做法不一样。一般而言，SPFA是队头扩展出的点加到最后面去，顺序扩展。这题有一个特殊性，在于边权只有两种0和1。如果尽量走$0$权边，更新出来的dist肯定较优，这意味着选择了$0$权边走到新的点，这个点以后被再更新的次数就变少了。

  所以想办法先做从$0$权边走到的点，后做从$1$权边走到的点。

  还是开一个队列，但队头指针跟队尾指针初始时指在中间位置，每次从$0$权边走的点，将队头往左移，每次从$1$权边走的点，将队尾往右移。队头被更新完以后往右移，进入下一次更新……

  这样优化的效果是惊人的。几乎秒出。

# tour

时间限制：$1000MS$   内存限制：$131072KB$  tour.cpp

### 题目描述

你赢得了一场航空公司举办的比赛，奖品是一张加拿大环游机票。旅行在这家航空公司开放的最西边的城市开始，然后一直自西向东旅行，直到你到达最东边的城市，再由东向西返回，直到你回到开始的城市。除了旅行开始的城市之外，每个城市只能访问一次，因为开始的城市必定要被访问两次（在旅行的开始和结束）。

给出城市的列表，和某些两城市之间的直达航线列表。求在满足条件的情况下访问的最多城市数。

### 输入格式

Line $1$: 航空公司开放的城市数 $N$ 和将要列出的直达航线的数量 $V$。$N$ 是一个不大于$ 100 $的正整数。$V $是任意的正整数

Lines $2\dots N+1$: 每行包括一个城市名称。城市名称按照自西向东排列。不会出现两个城市在同一条经线上的情况。每个城市的名称都是一个字符串，最多$15$字节，由大小写字母组成，城市名称中没有空格。

Lines $N+2\dots N+2+V-1$: 每行包括两个城市名称（由上面列表中的城市名称组成），用一个空格分开。这样就表示两个城市之间的直达双程航线。

### 输出格式

按照最佳路线访问的不同城市的数量 $M$。如果无法找到路线，输出$1$。

## 题解

由于题目告诉我们：只能从西到东走，然后从东到西返回。这样的限制很容易让我们想到多线程DP。
由于从东到西返回，等价于从西到东再走一条另外的路。所以，我们可以令$f(i,j)$表示第一次走到i，第二次走到j所走过
的最大城市数。
由于第一次路线和第二次路线没有差别，所以$f(i,j) = f(j,i)$，我们只考虑$i\le j$的情况(由于两条路线的出发点都是城市$1$，
所以要取等号)。
明显，$f(i,j)$可以转移给$f(i,k)$与$f(j,k)$，为了防止重复，$k>j$。
统计答案的时候，我们只需要找出最大的$f(i,n)$即可，但是$i$与$n$需要有连边，否则从$n$是回不到$i$的。



啊还有我的思路。用bitset状态压缩路径，用按位与为零表示来和去不是一个城市，有连边的话继承上一个城市的所有路径（在继承状态上按位或这个城市代表的$1$）。玄学复杂度A了。

			}
		}
	}
	if (x == 1) ans = 2;

}
