# 小Z爱求和

$sum.in/.out/.cpp$

## 【问题描述】

小 Z 有有一个长度为$n$序列$a_1,a_2,\dots,a_n$。
小 Z 想求
$$
\sum_{i=1}^n\sum_{j=1}^n\mbox{kth}_{\min}(\mbox\{a_i,a_{i+1},\dots,a_j\mbox\})+\mbox{kth}_{\max}(\mbox\{a_i,a_{i+1},\dots,a_j\mbox\})\mod 998244353
$$
其中$$\mbox{kth}_{\min}(S)$$为$S$集合中第$k$小的数字，若$|S|<k$，则$$\mbox{kth}_{\min}(S)=0$$；同理$$\mbox{kth}_{\max}(S)$$为$S$集合中第$k$大的数字，若$|S|<k$，则$$\mbox{kth}_{\max}(S)=0$$。

来帮帮他吧。

# 关押罪犯

$prison.in/.out/.cpp$

## 【问题描述】

B 城有一座监狱，一共关押着$N$名罪犯，编号分别为$1-N$。

他们的关系十分不和谐。很多罪犯之间甚至积怨已久，如果客观条件具备则随时可能爆发冲突。

在详细考察了$N$名罪犯间的矛盾关系后，警察局长发现罪犯之间的矛盾关系可以用一个$N$个点$M$条边的无向图来表示：如果$x$到$y$有一条边，表示罪犯$x$和罪犯$y$有矛盾。

现在警察局长要把这些罪犯分成一些小组，每名罪犯属于且仅属于一个小组。

为了开展活动顺利，要求每个小组内最多有$K$对罪犯有矛盾，同时为了管理方便，警察局长希望最小化分成的小组数量。

那么，应如何分配罪犯，才能使分成的小组数量最少？这个最小值是多少？

## 【输入格式】

输入文件名为prison.in。

第一行$3$个整数，$N,M,K$，意义见上述。

接下来$M$行，每行两个数字$x,y$。表示$x$和$y$之间有矛盾。

## 【输出格式】

输出文件名为prison.out。

输出一行一个值，表示最少的分组数量。

##【样例输入】

```
3 3 1
1 2
4
2 3
1 3
```

## 【样例输出】

```
2
```

## 【数据规模和约定】

对于每个测试点，$N$分别等于$2,4,6,8,10,12,14,15,16,16$。

$0 ≤ M,K ≤ \frac{N(N−1)}2$

保证是一个无自环、无重边的无向图。

## 做题

一看这个数据范围，能想到要状压DP。设$f[s]$为在$s$这种集合中最少分多少组，$s$对应位上为$1$表示这个罪犯已经被考虑过并扔进了组里。

然后先预处理，在读入无向边的时候处理出与$x$有矛盾的$y$的集合$to[s]$。

遍历状态$s$，若$j$没在$s$集合中，那么就考虑一种可能的转移

$$
son[s]=\min(son[s],son[s\mbox{ xor }(1<<j)]+cnt(to[j]\&s))
$$

这里采用了于文阔学长的一种巧妙解法，$cnt$函数就是数当前数里面有多少个二进制位是$1$，也就是加上多少对矛盾，如此就可以处理出所有可能的集合以及其中的矛盾数了。

这个异或有点迷，我还不太清楚到底什么意思，以及为什么会A掉。

然后设$f[s]$表示在$s$的情况下最少分多少组。当$son[s]\le k$的时候这些罪犯可以被分在一组里，所以$f[s]=1$。

设$s1$是$s$的子集，如果$s1$中的矛盾数不大于$k$的话，说明可以从$s-s1$的状态转移过来

$$
f[s]=\min(f[s],f[s-s1]+1)
$$

还有一个枚举子集的小技巧

```cpp
for (int s = 1; s < (1 << n); ++s) 
  for (int s1 = s; s1; s1 = (s1 - 1) & s) 
```

如此两重循环，就可以枚举$s$以及$s$的非空子集，若$s1$从$(s-1)\&s$开始的话就是非空真子集，如果要包含空集，判断条件中```s1```改为```~s1```即可。时间复杂度$\mathcal{O}(3^n)$。

```cpp
#include <bits/stdc++.h>
#define LOCAL LOCAL
using namespace std;

namespace STDquantum {

const int N = 17;
int n, m, k, son[1 << N], f[1 << N], to[1 << N];
inline int cnt(int s) {
  int ans = 0;
  while (s) {
    if (s & 1) {
      ++ans;
    }
    s >>= 1;
  }
  return ans;
}
inline void main() {
  scanf("%d%d%d", &n, &m, &k);
  for (int i = 1, x, y; i <= m; ++i) {
    scanf("%d%d", &x, &y);
    --x, --y;
    to[x] |= 1 << y, to[y] |= 1 << x;
  }
  memset(son, 0x3f, sizeof(son));
  son[0] = f[0] = 0;
  for (int s = 1; s < (1 << n); ++s) {
    for (int j = 0; j < n; ++j) {
      son[s] =
          min(son[s], son[s ^ (1 << j)] + cnt(to[j] & s));
    }
  }
  for (int i = 1; i < 1 << n; ++i) {
    if (son[i] <= k) {
      f[i] = 1;
    } else {
      f[i] = n;
    }
  }
  for (int s = 1; s < 1 << n; ++s) {
    for (int s1 = s; s1 > 0; s1 = (s1 - 1) & s) {
      if (son[s1] <= k) {
        f[s] = min(f[s], f[s - s1] + 1);
      }
    }
  }
  printf("%d", f[(1 << n) - 1]);
}

}  // namespace STDquantum

int main() {
#ifndef ONLINE_JUDGE
#ifdef LOCAL
  freopen("test.in", "r", stdin);
  freopen("test.out", "w", stdout);
#endif
#ifndef LOCAL
  freopen("T138969_Prison.in", "r", stdin);

#endif
  STDquantum::main();
  return 0;
}
```

# 小P走迷宫

$maze.in/.out/.cpp$

## 【问题描述】

小P和小T在一个$n×m$的网格图上走迷宫，这个网格图上有一些位置是陷阱，所以不能经过这些位置。

小P 和小T 都只通过向右或者向下走就从$(1,1)$出发走到$(n,m)$。

小T发现，他走过的路径是和小P走过的路径除了起点和终点外是不相交的。

现在小P想问问你，一共有多少种这样的走法。

简单来说，就是问在有坏点的网格图上只能向下或向右走，从$(1,1)$到$(n,m)$不相交的两条路径的方案数是多少？

## 【输入格式】

输入文件名为maze.in。

第一行两个正整数$n$和$m$，代表迷宫的大小。

接下来$n$行每行一个长度为$m$的$01$串，描述迷宫中的每一个点。

其中$0$代表可以走，$1$代表不能走。

数据保证迷宫的左上角和右下角可以走。

## 【输出格式】

输出文件名为maze.out。

一行一个整数，代表小P和小T左上角走到右下角的方案数，答案对$1000000007(10^9 + 7)$取模。

注意：对于两条不相交的路径$A$和$B$，如果小P走了$A$，小T走了$B$，和小P走了$B$，小T走了$A$我们视为同一种方案。

## 【样例输入】

```
4 4
0001
0100
0000
0000
```

## 【样例输出】

```
5
```

## 【数据规模和约定】

对于$20\%$的数据，$2 ≤ n,m ≤ 10$；

对于$40\%$的数据，$2 ≤ n,m ≤ 100$；

对于另外$10\%$的数据，迷宫中不存在陷阱(坏点)；

对于$100\%$的数据，$2 ≤ n,m ≤ 2000$。

## 做题

这题像极了找规律。虽然好像要用卡特兰数做，但我并不清楚怎么用。。。

是这样的，我们先读题，发现是找两条左上到右下不交的路径，坏点好解决，先不用管。这样的路径呢如下图所示，实际是要沿着网格走的，但是领会精神很重要。

{% asset_image 1.png 图一 %}

首先一个人要从$(2,1)$到$(n,m-1)$，同时另一个人从$(1,2)$到$(n-1,m)$，否则没办法保证路径不相交，其次呢，一个人这么走，另一个人那么走，方案数就是两个人分别走的路径条数的乘积。我们发现这样走的话还是有可能有相交的路径。这时候就要考虑怎么去掉那些相交的路径。

有一个结论挺有用的，就是交换两个人的终点，如下图所示。

{% asset_image 2.png 图二 %}

交换后两个人如图走的路径数，就是会有交点的路径数。

光给出这个结论似乎不是很明显，下面来简单地证一下。

首先，这样的两条路径一定有至少一个交点。

把两个人路径的到右下角最后一个交点找到，因为路径可以不止有一个交点。然后将两个人交点到终点的路径交换一下，如下图

{% asset_image 3.png 图三 %}

这样做有什么好处呢？就是两个人的终点又回到了先前的样子，而之前的终点正是我们需要的，而且这样交换后中间还是会有至少一个点相交或重叠。总的来说，就是正确的起点走向正确的终点，而且中间路径还相交。

可以发现这就是我们想要的需要排除的部分了。而且任意两条相交的路径通过交换最后一部分交换终点，还能保证图二的走法与想要排除的图一的走法一一对应，于是图一统计的答案减去图二统计的答案就能得到最终答案。

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace STDquantum {

const int N = 2001, mod = 1e9 + 7;
typedef long long ll;
int n, m;
ll ans, f[N][N];
char s[N][N];
ll sol(ll x1, ll y1, ll x2, ll y2) {
  memset(f, 0, sizeof(f));
  f[x1][y1] = 1;
  for (register int i = x1; i <= x2; ++i) {
    for (register int j = y1; j <= y2; ++j) {
      if (s[i][j] == '1') continue;
      (f[i][j] += f[i - 1][j] + f[i][j - 1]) %= mod;
    }
  }
  return f[x2][y2];
}
inline void main() {
  scanf("%d%d", &n, &m);
  for (int i = 1; i <= n; ++i) {
    scanf("%s", s[i] + 1);
  }
  ans = sol(1, 2, n - 1, m) * sol(2, 1, n, m - 1) -
        sol(2, 1, n - 1, m) * sol(1, 2, n, m - 1);
  printf("%lld", (ans % mod + mod) % mod);
}

}  // namespace STDquantum

int main() {
  STDquantum::main();
  return 0;
}
```
