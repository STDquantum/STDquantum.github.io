# 3460 蓝蓝的棋盘

这题酷似原题诶

## 题目描述

淘淘和蓝蓝在下棋.

这个棋盘是1xn的,棋盘的第i个格子上有一个数a[i],因此我们可以把棋盘看做一个序列.一开始棋子在位置0,双方得分都是0.

**双方轮流操作棋子**,如果当前棋子的位置是p,可以选择把棋子移动到[p+1,min(n,p+m)]的范围内,即**最多往右移动m格,但是不能移出棋盘**.

每次操作之后,当前移动棋子的人会获得移动到的格子上数字的分数.

当棋子被移动到位置n的时候,游戏就结束了，但是可以取得位置n的分数

淘淘和蓝蓝都会按照自己的最优策略移动棋子.蓝蓝想知道,当他先手的时候,**他的分数减去淘淘的分数**是多少. （最优策略指最大化自己的得分减对方的得分）

## 输入

第1行,两个整数n和m,用空格隔开.

第2行,n个整数,A[1] .. A[n],用空格隔开.

## 输出

一行一个整数表示答案.

## 样例输入输出

### 【样例输入1】

```
1 9 
1
12
```

### 【样例输出1】

```
1
1
```

### 【样例输入2】

```
5 3
-1 5 -9 8 1
12
```

### 【样例输出2】

```
-2
1
```

### 【样例输入3】

```
10 3
1 -3 4 -7 22 -12 34 -4 -7 -9
12
```

### 【样例输出3】

```
14
```

## 提示

在样例1中,蓝蓝只能移动到第1个格子,答案为1-0=1

在样例2中,蓝蓝先移动到2,淘淘移动到4,蓝蓝再移动到5.棋子的移动路径是0->2->4->5.

在样例3中,棋子的路径是0->3->5->7->9->10.

**下发文件中有两个大样例** **chess-sample4.in/out** **和** **chess-sample5.in/out**

[/upload/file/20200719/20200719101045_80834.rar](https://www.qizhishu.com/upload/file/20200719/20200719101045_80834.rar)

对于30%的数据,n≤15;

对于80%的数据,m≤100;

对于100%的数据,1≤n≤500000,1≤m≤998244353,-998244353≤a[i]≤998244353.

## 要求

时间限制：1.0Sec  内存限制：128MB

## 题解

- 子任务1：n≤15
  - 直接枚举怎么移动。反正时间复杂度大得很，数据范围小得很。
- 子任务2：m≤100
- 子任务3：无特殊限制
  - 这 $m$ 让人一看就知道是单调队列优化。考虑 $f[i]$ 表示到 $i$ 这个位置时，先手能比后手多的最大值。
  - 考虑转移：首先先后手是可以相对变化的，所以只需要考虑每个位置的最优就好了。自己优的话相当于对方劣，也就是让对方的值最小。也就是 $f[i]=\max(a[j]-f[j])$，也就是在 $j$ 这个位置拿到最优的分数。
  - 因为结束点在 $n$ ，不能超出去，所以倒着枚举加入队列就行。

```cpp
#include <bits/stdc++.h>
#define LOCAL
using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  int f = 1;
  while (ch = getchar(), !isdigit(ch)) {
    if (ch == '-') f = -1;
  }
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
  x *= f;
}

typedef long long ll;
using namespace std;
const int N = 2e6 + 10;
ll n, m, a[N], f[N], q[N], cur[N];
inline void main() {
  read(n), read(m);
  for (ll i = 1; i <= n; i++) read(a[i]);
  ll head = 1, tail = 0;
  q[tail] = a[n] - f[n], cur[tail] = n;
  for (ll i = n; ~i; --i) {
    while (head <= tail && cur[head] - i > m) ++head;
    f[i] = q[head];
    /* 注意这个地方不要太着急弹出队尾，因为队列里都是合法的，提前弹出可能导致较优的解无法选中 */
    /* 毕竟自己不参与这次f数组更新的竞争，要是还插一脚的话就不好了 */
    while (head <= tail && a[i] - f[i] >= q[tail]) --tail;
    q[++tail] = a[i] - f[i], cur[tail] = i;
  }
  printf("%lld", f[0]);
}

}  // namespace STDquantum

int main() {
#ifndef ONLINE_JUDGE
#ifdef LOCAL
  freopen("test.in", "r", stdin);
  freopen("test.out", "w", stdout);
#endif
#ifndef LOCAL
  freopen("QZ3460_Blue_Chess.in", "r", stdin);
  freopen("QZ3460_Blue_Chess.out", "w", stdout);
#endif
#endif
  STDquantum::main();
```

# 3461 淘淘的集合

## 题目描述

淘淘有n个数字a[1]...a[n]和n个集合,一开始所有的a[i]都是0,第i个集合初始为{i}.

淘淘有时候会合并两个集合或者给一个集合里的所有数整体加上一个值.

蓝蓝觉得淘淘的数字太大了,有时会把a[l..r]都赋值为0,有时会询问一段区间的数字的和

形式化的讲,就是有四种事件会发生:

1 x y合并数字x所在的集合和y所在的集合.如果x和y已经在同一个集合,则没有变化;

2 x v设x所在的集合为S，对于任意y∈S(y∈S指y在集合S里),a[y]+=v

3 l r 把a[l]...a[r]都赋值为0;

4 l r 询问a[l]...a[r]的和.

一共会发生m次这样的事件,你能正确解答蓝蓝的每次询问吗?

## 输入

第1行,两个整数n和m,用空格隔开.

第2~(m+1)行,每行三个整数表示一个操作.

**保证事件4的(r-l+1)的和不超过10000000.**

## 输出

对于每个事件4,输出一行一个整数表示询问的答案.

## 样例输入

```
3 10
2 1 -7
1 1 3
2 3 5
4 1 3
3 2 3
4 1 3
2 3 -7
4 1 3
2 2 2
4 1 3
```

## 样例输出

```
3
-2
-16
-14
```

## 提示

对于30%的数据,n,m≤3000;

对于另外10%的数据,没有事件1;

对于另外20%的数据,没有事件3;

对于另外20%的数据,保证事件3的(r-l+1)的和不超过10000000.

对于100%的数据,1≤n≤200000,1≤m≤50000;事件1,2保证1≤x,y≤n,-998244353≤v≤998244353;事件3,4保证1≤l≤r≤n;**保证事件4的(r-l+1)的和不超过10000000.数据保证所有事件4的答案不超过C++ long long的范围.**

## 要求

时间限制：1.0Sec  内存限制：1024MB

## 题解

我们先一点一点考虑：

- 子任务1：$n,m\le3000$
  - 这个直接暴力做就好了，复杂度完全过得去。
- 子任务2：没有事件 $1$
  - 事件 $1$ 是集合操作，而除了 $2$ 对集合操作以外都是单点、区间操作。没有了 $1$，事件 $2$ 也就变成了单点操作。
  - 单点加、区间赋值、区间求和，这变成了一个线段树问题。
- 子任务3：没有事件 $3$
  - 因为事件 $4$ 总求和下标个数不超过 $10^7$，也就是可以变为规模为 $10^7$ 的单点查询。
  - 而事件 $1,2$ 的集合操作，可以通过并查集实现。
  - 整体加：在并查集上打标记即可。
  - 合并：并查集启发式合并，暴力更改两个集合中集合内数的数量比较少的集合（简称小集合）中的点的权值，让它增加 $tag[SmallSet] - tag[LargeSet]$，也就是说，把小集合的节点都正确改正后，为了适应要合并进去的大集合，再减去大集合的标记，这样便可以与大集合的标记保持统一（也就是查询时候加上 $tag[LargeSet]$ 的结果正确）。
  - 单点查询：把当前数的权值加上当前集合的标记。

- 子任务4：无特殊限制
  - 考虑怎么实现区间的赋值。
  - 利用并查集启发式合并时的思想，为了消除赋值 $0$ 所带来的影响，直接把当前要修改的点能影响到的所有查询的答案都减去当前的值，这样数组没变，并查集该怎么操作就怎么操作，只是查询的时候消去影响。
  - 但是 $3$ 操作的时候还不清楚之后会影响哪些查询，所以先将查询和区间赋值操作预处理。
  - 这样又出现了一个问题：如何得知之前最近的一次赋值为 $0$ 的位置？
  - $10^7$ 规模，查询要做到 $O(1)$，还要区间修改，会想到分块。分块维护一下最近一次赋值为 $0$ 的时间，在这个时间放一个信号要减去这个时间对应的值就好了。

```cpp
#include <bits/stdc++.h>
#define LOCAL
using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  int f = 1;
  while (ch = getchar(), !isdigit(ch)) {
    if (ch == '-') f = -1;
  }
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
  x *= f;
}

typedef long long ll;
const int N = 5e5 + 10, M = 1e7 + 10;

struct K { // 分块
  int n, m, s, cur[N], l[N], r[N], a[N], tag[N];
  inline void init(int x) {
    n = x, s = sqrt(x) + 1;
    for (int i = 1; i <= n; ++i) cur[i] = (i - 1) / s + 1;
    m = cur[n];
    for (int i = 1; i <= m; ++i) {
      l[i] = (i - 1) * s + 1, r[i] = min(n, i * s);
      tag[i] = -1;
    }
  }
  inline void pushdown(int k) {
    if (tag[k] != -1) {
      for (int i = l[k]; i <= r[k]; ++i) a[i] = tag[k];
    }
    tag[k] = -1;
  }
  inline void add(int L, int R, int w) {
    int k;
    if (cur[L] == cur[R]) {
      pushdown(cur[L]);
      for (int i = L; i <= R; ++i) a[i] = w;
      return;
    }
    pushdown(k = cur[L]);
    for (int i = L; i <= r[k]; ++i) a[i] = w;
    pushdown(k = cur[R]);
    for (int i = l[k]; i <= R; ++i) a[i] = w;
    for (int i = cur[L] + 1; i < k; ++i) tag[i] = w;
  }
  inline int query(int k) { // 输出前一个赋为0的时间
    return ~tag[cur[k]] ? tag[cur[k]] : a[k];
  }
} K;

struct UF { // 并查集
  int fa[N];
  ll tag[N], val[N];
  vector<int> g[N];
  inline void UFset(int n) {
    for (int i = 1; i <= n; ++i) {
      fa[i] = i, tag[i] = val[i] = 0, g[i].clear();
      g[i].push_back(i);
    }
  }
  inline int Find(int x) {
    return x == fa[x] ? x : (fa[x] = Find(fa[x]));
  }
  inline void Union(int x, int y) {
    x = Find(x), y = Find(y);
    if (x == y) return;
    if (g[x].size() < g[y].size()) swap(x, y);
    for (int i = 0, sz = g[y].size(); i < sz; ++i)
      val[g[y][i]] += tag[y] - tag[x], g[x].push_back(g[y][i]);
    tag[y] = 0, fa[y] = x, g[y].clear();
  }
  inline ll query(int x) { return val[x] + tag[Find(x)]; }
  inline void add(int x, int w) { x = Find(x), tag[x] += w; }
} UF;

int cnt1, cnt2, op[N], opx[N], opy[N];
struct Q1 {
  int id, x; // 消去操作
} q1[M]; // 注意数组要开足够大
struct Q2 {
  int id, t, l, r; // 离线查询
} q2[N];
vector<Q1> g[N];

int n, m;
ll ans[N];
inline void main() {
  read(n), read(m);
  K.init(n);
  for (int i = 1; i <= m; ++i) {
    read(op[i]), read(opx[i]), read(opy[i]);
    if (op[i] == 3) K.add(opx[i], opy[i], i);
    if (op[i] == 4) {
      q2[++cnt2] = (Q2){cnt2, i, opx[i], opy[i]}; // 离线查询操作
      for (int j = opx[i]; j <= opy[i]; ++j) {
        q1[++cnt1] = (Q1){cnt2, j}; // 表示在cnt2这个查询里找j的权值
        g[K.query(j)].push_back(q1[cnt1]); // 在j上一次被赋零的时间插入一个消去操作
      }
    }
  }
  int cur = 1;
  UF.UFset(n);
  for (int i = 1; i <= m; ++i) {
    if (op[i] == 1) UF.Union(opx[i], opy[i]);
    if (op[i] == 2) UF.add(opx[i], opy[i]);
    for (int j = 0, sz = g[i].size(); j < sz; ++j)
      ans[g[i][j].id] -= UF.query(g[i][j].x); // 消去影响
    while (cur <= cnt2 && q2[cur].t <= i) { // 完成i之前的查询
      for (int j = q2[cur].l; j <= q2[cur].r; ++j)
        ans[q2[cur].id] += UF.query(j);
      ++cur;
    }
  }
  for (int i = 1; i <= cnt2; ++i) printf("%lld\n", ans[i]);
}

}  // namespace STDquantum

int main() {
#ifndef ONLINE_JUDGE
#ifdef LOCAL
  freopen("D:\\OI\\STDquantum\\test.in", "r", stdin);
  freopen("D:\\OI\\STDquantum\\test.out", "w", stdout);
#endif
#ifndef LOCAL
  freopen("QZ3461_Tao_Set.in", "r", stdin);
  freopen("QZ3461_Tao_Set.out", "w", stdout);
#endif
#endif
  STDquantum::main();
  return 0;
}
```

# 3462 蓝蓝的程序

## 题目描述

一天,蓝蓝写了一段代码并用它们实现了一些功能.

但是这些程序跑的非常慢,不能在比赛的(3.5小时/5小时)中获得结果.

请你帮助他优化这些程序,得到正确的结果.

一共有10个程序。(10个子任务)

因为输入文件太大，所以我们不提供输入文件，只会提供数据规模n和一个数seed,选手可以用seed和每道题给你的gen1~gen10.cpp来生成输入数据。

seed和每道题的n在选手目录下的1.txt~10.txt中.

蓝蓝提供了他写的程序1.cpp~10.cpp,保证它们一次运行只会输出一个数。

蓝蓝的代码不能直接运行,因为它只能在规定时间内解决小范围的数据。

你会发现，在蓝蓝的程序里，输出的数对998244353取模。

我们要你求出问题的输出。

为了方便选手检查代码，我们会提供答案对19260817取模的数值，存放在选手目录下的1.ans~10.ans中。

**因为某些神秘的原因,我们不使用提交答案式的评测,而是每个测试点里输入一个数Num表示子任务编号,然后你在程序中输出答案.**

[/upload/file/20200719/20200719102119_62878.rar](https://www.qizhishu.com/upload/file/20200719/20200719102119_62878.rar)

## 输入

一行一个数Num表示子任务编号. 

## 输出

一行一个整数表示这个测试点的答案.

## 要求

时间限制：1.0Sec  内存限制：128MB

## 题解

好像之前考过的一道题啊。。<a href="../20200410-mo-ni-sai-ri-ji" title="20200410模拟赛日记">NPIO十合一</a>。

- 子任务1：1e9求和、最大、最小值

  一边造数据一边查就行

- 子任务2：4e4见方的矩阵，求前缀和（左上角部分）的和。

  考虑每个数字 $a[i][j]$ 对答案的贡献，不难发现是$(n-i+1)*(n-j+1)*a[i][j]$。

  那么就可以不开数组完成了。程序可以在 1min 内跑完。

  另解：不考虑贡献，滚动数组优化掉计算前缀和的空间复杂度，直接模拟程序的计算。空间复杂度为 $O(n)$。

- 子任务3：4e4见方的矩阵，求前缀和的前缀和的和。

  仍然可以考虑每个数字 $a[i][j]$ 对答案的贡献，记 $f(n) = n*(n+1)/2$，不难发现 $a[i][j]$ 对答案的贡献为 $f(n-i+l)*f(n-j+ 1)* a[i][j]$。那么就可以不开数组完成了。

  另解：不考虑贡献，滚动数组优化掉计算前缀和的空间复杂度，直接模拟程序的计算。空间复杂度为 $O(n)$。

- 子任务4：1e15的0次方和

  直接输出 $10^{15}\bmod p$

- 子任务5：5e15的1次方和

  等差数列求和，$\frac{n(n+1)}{2}\bmod p$

- 子任务6：5e15的2次方和

  有$(n+1)^2 = n^2+2n+1$，则
  $$
  n^2-(n-1)^2=2(n-1)+1\\
  \vdots\\
  2^2-1^2=2\times1+1
  $$
  左边全部加起来，右边全部加起来，得到$n^2-1=2(1+2+\cdots+n)+n\times1=2\sum\limits_{i=1}^ni^1+n$，之后再算的时候，就可以把上一问求出的$\sum\limits_{i=1}^ni^1$带入这个式子，便得到了通项。

  $ans=\frac{n(n+1)(2n+1)}6\bmod p$

- 子任务7：5e15的3次方和

  求法同上，$ans=\left(\frac{n(n+1)}{2}\right)^2\bmod p$

- 子任务8：5e15的4次方和

  求法同上，$ans=\left(\frac{n (1 + n) (1 + 2 n) ( 3 n + 3 n^2 -1)}{30}\right)\bmod p$

  插播一句，Python大法好啊，自带高精

- 子任务9：
