# 3479 climb

## 样例输入

```
4 3
1 7 1
1 5 0
2 1 0
```

## 样例输出

```
0
7
3
6
```

## 解题

考虑树形DP，$f[i]$ 表示从 $i$ 开始到达山顶的最短时间。起始点就是 $f[1]=0$，然后从 $1$ 开始遍历整棵树。

如果 $i$ 这里有缆车，那么所有 $dep$ 小于等于 $dep_i$ 的点都可以转移。

小于 $dep_i$ 的好说， $f[i]=\min\{f[j]+|dep_i-dep_j|\times k\}$。

等于 $dep_i$ 的，意味着有缆车的点的值还可以被同层所有点的DP值更新。

先考虑所有点都可以被同层更新的情况（任务点 $5,6$），也就是把层作为转移的依据。

首先一层中若一个点是最小值，在下面层转移的时候走到这里肯定比同层的点要优。

每多跨一层，它的值就增加 $k$，而且是每一层均增加，所以如果一层的值在加上或减去 $n\times k$ 后没有其它层优，那么之后的转移它也不会更优。

所以可以考虑维护一个点，它是前 $dep_{i-1}$ 层的最优决策点，并记录它的 $dep$，这样在每一层转移的时候均是 $O(1)$ 的。

现在再把不能使用缆车的条件加进来，也就是说这个点只能走路到它的父亲节点，那就直接 $f[i]=f[fa]+w$。先处理掉不能用缆车的节点，同时更新 $dep_i$ 层的决策点。~~最后可以用缆车的点按照前面的方法直接转移到决策点即可~~。千万别！可以用缆车也不一定要用，还可以直接步行。每个点都更新一遍最优决策点，然后重新跑一遍这一层可以走缆车的节点以防在这个点后面更新决策点。

时间复杂度 $O(n)$ 左右。

using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  while (ch = getchar(), !isdigit(ch)) {}
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
}

const int M = 5e5 + 10;
int n, k, maxx, ban[M], dep[M], f[M], fa[M], val[M];
vector<int> g[M];
int cur = 1;
void bfs() {
  for (int i = 1, v; i <= maxx; ++i) {
    for (int j = 0, sz = g[i].size(); j < sz; ++j) {
      v = g[i][j], f[v] = f[fa[v]] + val[v]; // 一定要首先从父节点走！
      if (!ban[v]) { // 没被ban掉缆车就能用上面的最优决策点转移
        f[v] = min(f[v], (dep[v] - dep[cur]) * k + f[cur]);
      }
      if (f[v] < f[cur] + k * (dep[v] - dep[cur])) { cur = v; } // 更新最优
    }
    for (int j = 0, sz = g[i].size(); j < sz; ++j) {
      if (!ban[v = g[i][j]]) { // 没被ban掉还可以从同层最优决策点转移，再来一遍
        f[v] = min(f[v], (dep[v] - dep[cur]) * k + f[cur]);
      }
    }
  }
}
inline void main() {
  read(n), read(k);
  for (int i = 2; i <= n; ++i) {
    read(fa[i]), read(val[i]), read(ban[i]);
    /* 一开始写了个dfs后来发现不用，保证fa<i了就直接更新就行 */
    dep[i] = dep[fa[i]] + 1, maxx = max(maxx, dep[i]);  
    g[dep[i]].push_back(i);
  }
  bfs();
  for (int i = 1; i <= n; ++i) { printf("%d\n", f[i]); }
}

}  // namespace STDquantum

# 3480 h

## 题目描述

经过不懈的努力，小Z的团队取得了突破性进展。

小Z的团队有一个碰撞装置。在内部有一个数轴，上面有n个样本，第i个样本现在在pi位置，速度是vi（速度的正负代表不同的方向）。在U时间后，第i个样本就会移动到pi+U*vi上。

而这个碰撞装置能够运行，当且仅当没有任何样本进行碰撞（即移动到同一点上）。如果没有碰撞，那么它就会一直运行下去。

为了知道碰撞装置的最高效率，小Z允许拿走k个样本。那么，拿走k个样本后碰撞装置最长能运行多久？

## 输入

第一行两个整数n，k，用空格隔开。

接下来n行，每行两个整数p，v，表示每个样本的初始位置和速度，用空格隔开。

## 输出

一行一个实数表示答案，精确到小数点后4位。

注意，如果样本永远不会碰撞，输出“Forever”。（不要输出双引号）

## 样例输入

### 1

```
4 1 
1 1 
3 -1 
5 2 
7 -2
```

```
1.0000
```

### 2

```
4 2 
1 1 
3 -1 
5 2 
7 -2
```

```
Forever
```

### 3

```
5 0
2 3
233 -9
55 1
66 -2
1000 -10
```

```
3.6667
```

## 提示

【样例1解释】

把第三或第四个样本拿走，那么第一、第二个样本会在1单位时间后碰撞。

【样例2解释】

把第二个、第四个样本拿走。

【提示】

如果你是C++选手，最后输出答案可以使用`printf(“%.4f\n”,ans)`或`cout<<fixed<<setprecision(4)<<ans<<endl`，但需要注意的是，用了`scanf`、`printf`就不要使用`cin`、`cout`(同时不要去同步)，用了`cin`、`cout`就不要使用`printf`、`scanf`。**不然后果自负**。

【数据范围】

对于前30%的数据，n=2。

对于前60%的数据，k=0。

对于前90%的数据，n<=1000。

对于100%的数据，n,k<=50000，所有的位置和速度在[-1000000,1000000]中。保证刚开始不会有样本在同一位置。

## 解题

- 前 $30\%$：$n=2$

  让第一个块相对第二个块运动。

  第一个块的坐标减去第二个块的坐标，速度（注意是速度不是速率）减去第二个块的速度。

  这样就相当于第二个块在原点待着不动，第一个块来回跑，这样问题不就简单多了吗？

  如果坐标和速度符号相同或者速度为零（坐标不能为 $0$，因为保证了两个块不放一块），那么永远不可能相遇，反之输出坐标除以速度的相反数就可以，因为这样一定能保证结果是正的。

- 前 $60\%$：$k=0$

  因为 $n\le1000$，所以考虑 $O(n^2)$ 的做法。

  直接枚举两个块，用随便什么方法判断会不会撞上，如果会撞上就记录一个最小值，然后输出就行。

- 前 $90\%$：$n\le 1000$

  考虑二分答案，如果能做到 $O(n^2)$ 判断就能过。

  我们来观察数据，先把数组按照横坐标排序，二分一个时刻 $t$，然后令 $x_i$ 为横坐标， $v_i$ 为速度，现在若 $x_i+v_i\times t\ge x_j+v_j\times t\ (i<j)$，那么这两块一定会撞在一起，若把 $x_i+v_i\times t$ 记为 $a_i$ 的话，这就变成了一个逆序对。
  
  我们来看对于给出的 $k$ 当前的合法状态是什么。所有块不能撞在一起，也就是说不能存在逆序对。我们去掉了 $k$ 个点后得到了一个长度为 $n-k$ 的单调上升子序列。那么也就是说，如果 $a$ 数组的最长上升子序列是不小于 $n-k$ 的，该时间点就不会有块撞上。
  
  问题转化为了求一个数组的最长上升子序列，经典的DP做法是 $O(n^2)$ 的，可以通过前 $90\%$。

- 前 $100\%$：无特殊限制

  最长上升子序列有 $O(n\log n)$ 的做法（贪心二分或树状数组）呀，总时间复杂度 $O(n\log^2n)$，可以通过本题。

#include <bits/stdc++.h>
using namespace std;

template <typename T> inline void read(T &x) {
  char ch;
  int f = 1;
  while (ch = getchar(), !isdigit(ch)) {
    if (ch == '-') f = -1;
  }
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
  x *= f;
}

namespace Brutalforce {

const int N = 5e5 + 10;
struct E {
  int x, v;
} e[N];
inline int sgn(int x) {
  if (x < 0) return -1;
  if (x == 0) return 0;
  return 1;
}
int n, k;
inline void main() {
  read(n), read(k);
  for (int i = 1; i <= n; ++i) { read(e[i].x), read(e[i].v); }
  if (n == 2) {
    e[1].x -= e[2].x, e[1].v -= e[2].v;
    if (sgn(e[1].x) == sgn(e[1].v) || !e[1].v) {
      puts("Forever");
    } else {
      printf("%.4lf\n", 1.0 * e[1].x / (-e[1].v));
    }
    return;
  }
}

}  // namespace Brutalforce

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  int f = 1;
  while (ch = getchar(), !isdigit(ch)) {
    if (ch == '-') f = -1;
  }
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
  x *= f;
}

const int N = 5e5 + 10;
const long double inf = 1e12, eps = 1e-6;
struct E {
  int x, v;
  inline friend bool operator<(const E &a, const E &b) {
    return a.x < b.x;
  }
} e[N];

int n, k;
long double a[N], b[N];
inline bool check(long double cur) { // 求最长上升子序列的过程
  int cnt = 1;
  for (int i = 1; i <= n; ++i) { a[i] = e[i].x + cur * e[i].v; }
  b[cnt] = a[1];
  for (int i = 2; i <= n; ++i) {
    if (a[i] > b[cnt]) {
      b[++cnt] = a[i];
    } else {
      *lower_bound(b + 1, b + cnt + 1, a[i]) = a[i];
    }
  }
  if (cnt >= n - k) return 1;
  return 0;
}
inline void main() {
  read(n), read(k);
  for (int i = 1; i <= n; ++i) { read(e[i].x), read(e[i].v); }
  sort(e + 1, e + n + 1);
  long double l = 0, r = inf, mid;
  while (abs(l - r) > 1e-8) {
    mid = (l + r) / 2;
    if (check(mid)) {
      l = mid;
    } else {
      r = mid;
    }
  }
  if (abs(mid - inf) < eps) {
    puts("Forever");
  } else {
    printf("%.4Lf", mid);
  }
}

}  // namespace STDquantum

  return 0;
}

# 3481 pancake

## 题目背景

{% asset_image 1.jpg %}

## 题目描述

巧克力薄饼是DoubleDuck山有名的异域小吃。淘淘到山顶时，薄饼店前已经排起了长长的队伍。

在等待的时候，淘淘拿出手机，进行了网上点餐。刚开始，他点了 $n$ 条巧克力薄饼，每个薄饼的长度为$a_i$（均为正整数）。点餐时还有一个奇怪的要求：对于每个长度为$len$的薄饼，它最多被切成$len$条长度均为正整数的薄饼，并且所有薄饼必须<u>总共恰好切</u>$k$<u>刀</u>。形式化地，对于长度为$n$的正整数序列$a_1, \dots, a_n$，应选择它们的一个拆分序列$\{b_{1,1},\dots, b_{1,k_1}\},\dots,\{b_{n,1},\dots,b_{n,k_n}\}$，满足所有数字都是正整数，并且
$\sum_{i=1}^n(k_i-1)=k$，$\sum_{i=1}^{k_i}b_{pos,i}=a_{pos}$。

淘淘当然不喜欢特别长的巧克力薄饼，因为里面夹的香蕉或草莓很容易就会掉下来。对于最后获得的巧克力薄饼，淘淘定义它的代价为所有长度的平方和，即
$\sum b^2_i$
。每次淘淘都会选择代价最小的一张方案进行点餐。

但在等待的过程中，淘淘会有$q$个想法。每个想法要么是将$k$增加$1$，要么是将$k$减少$1$，要么是多订一个长度为$x$的巧克力薄饼。

淘淘想要知道，他一开始和每个想法过后，最后的代价最小是多少。

## 样例输入

```
1 3 1
4
1
3
4
2
```

## 样例输出

```
8
6
16
24
```

## 解题

- 子任务1：$n=1,a_1\le15$

  考虑一个DP，设 $f[i][j]$ 表示切到 $i$，切了 $j$ 次的最小代价，转移：
  $$
  f[i][j]=\min_{o<i}\left\{f[o][j - 1]+(i-o)^2 \right\}
  $$
  因为数据范围很小，所以 $O(n^2k)$ 暴力枚举就行。

  答案是 $f[a_1][k]$。

- 子任务2：$k=0$

  一刀也不能切，直接每块的平方加起来。

- 算法 $1$

  可以发现如果把一个长度为 $len$ 的饼分为 $k$ 份，最优的做法是分成 $k-len\bmod k$ 份长度为 $\lfloor\frac {len}k\rfloor$ 的，还有 $len\bmod k$ 份长度为 $\lfloor\frac {len}k\rfloor+1$ 的。贪心的思想，最优分法下不存在两个差超过 $2$ 的划分。
  
  至于这个式子怎么推出来的，首先肯定和 $\lfloor\frac {len}k\rfloor$ 有关，毕竟这是一个平均数。其次就是让总体尽可能平均，若 $len$ 能整除 $k$，那自然很好，但是如果不能整除，就要优先给 $\lceil\frac {len}k\rceil$ 分配。一个 $len$ 只能塞下 $len\bmod k$ 个 $\lceil\frac {len}k\rceil$，多了份不够，剩下的就给 $\lfloor\frac {len}k\rfloor$ 了。
  
  举个例子吧，$14$ 分 $8$ 份。
  
  分成七个 $2$ 肯定是很优的（因为尽可能平均），但是最多放 $6$ 个，因为剩下两块要给两个 $1$。
  
  这样我们可以设 $f[i][j]$ 表示前 $i$ 个数中一共切了 $j$ 刀（注意切 $j$ 刀代表分成 $j+1$ 份）的最小代价，$p[i][j]$ 表示长度 $i$ 切 $j$ 次的最小代价，枚举 $o$ 转移：
  $$
  f[i][j]=\min_{0\le o\le \min\{k,len_i-1,j\}}\{f[i-1][j-o]+p[len_i][o]\}
  $$
  总共 $O(n^2k)$，能解决数据范围较小的全部询问。

- 算法 $2$

  上面那思路已经到极限了，要考虑一个更可的计划。

  对于一个 $p[i][j]$，随着 $j$ 的增大，$p[i][j]-p[i][j+1]$ 的值减小得很快，很快就会到达一个下界，而且值不升。类似反比例函数的样子。

  我们对于询问部分，考虑一个可以反悔的堆，也就是维护一个小根堆，堆里放 $p[i][j+1]-p[i][j]$，每次操作时取出堆顶累计进答案（这是个负的，小根堆也就是这个原因，让能减少的代价最大，也就是值最小），然后再放入 $p[i][j+2]-p[i][j+1]$，如果后面 $j$ 的位置上的值大于 $k$ 那就把这个值设为无穷大（也就是永远不会拿出来）。

  一开始的时候，把所有 $p[i][0]$ 累计进答案，并在这个堆里放入 $p[i][1]-p[i][0]$，之后做 $k$ 次取出的操作，就可以得到最优的解。

  对于询问 $1$，可以用此方法解决。询问 $2$ 的话，由于是减小 $k$，也就是增加代价。我们就维护另外一个删除用的反悔小根堆，在堆顶放入 $p[i][j-1]-p[i][j]$，使用时取出堆顶就行。

  一开始 $O(n\log n)$ 加入所有元素，$O(k\log n)$ 处理还没询问时候的答案，$O(q\log n)$处理每个询问。堆中元素拿出一个放进去一个始终是 $O(n)$ 的。

  总复杂度 $O((q+n+k)\log n)$ （直接不考虑 $3$ 询问的复杂度），可以解决数据范围较大的 $1,2$ 询问。

  其实在加入新元素的时候，直接将新元素的贡献放入堆中，然后 $k$ 减一再加上一，如果 $ans$ 没变的话就说明没贡献（雾

  可以证明所有 $3$ 操作一共是 $O(n\log n)$ 的，因为一个元素能提供的贡献就像反比例函数，减小得很快，很快就不优了。

using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  int f = 1;
  while (ch = getchar(), !isdigit(ch)) {
    if (ch == '-') f = -1;
  }
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
  x *= f;
}

typedef long long ll;
const ll N = 1e6 + 5, inf = 0x3f3f3f3f3f3f3f3f;
ll n, q, ans, m, a[N], k[N];
inline void write(ll x) {
  if (x < 0) putchar('-'), x = -x;
  if (x > 9) write(x / 10);
  putchar(x % 10 + '0');
}
inline ll get(int x, int k) {
  if (x < k || k == 0) return inf;
  ll i = x / k;
  return i * i * (k - x % k) + (i + 1) * (i + 1) * (x % k);
}
struct E {
  int id;
  ll w;
  E(int a = 0, ll b = 0) : id(a), w(b) {}
  inline friend bool operator<(const E &a, const E &b) {
    return a.w == b.w ? a.id < b.id : a.w < b.w;
  }
};
multiset<E> q1, q2;
inline void addK() {
  multiset<E>::iterator p = q1.begin();
  E A = *p;
  ans += A.w;
  int id = A.id;
  q2.erase(
      q2.find(E(id, get(a[id], k[id] - 1) - get(a[id], k[id]))));
  q1.erase(p);
  ++k[id];
  q1.insert(E(id, get(a[id], k[id] + 1) - get(a[id], k[id])));
  q2.insert(E(id, get(a[id], k[id] - 1) - get(a[id], k[id])));
}
inline void remK() {
  multiset<E>::iterator p = q2.begin();
  E A = *p;
  ans += A.w;
  int id = A.id;
  q1.erase(
      q1.find(E(id, get(a[id], k[id] + 1) - get(a[id], k[id]))));
  q2.erase(p);
