# T1

大细节题。

首先考虑把所有元素都上取整（下取整也可做，看完这篇后欢迎思考一下！），然后考虑怎么去掉多出来的那些贡献。

一个元素上取整和下取整之间顶多差 $1$，一共要改变 $len$ 个元素（长度当成 $2\times len$），要尽可能让答案最小。

前缀和求出 $sumTop$ ：区间元素上取整的和、$sumA$ ：区间所有元素的和、$sumNInt$：区间内不是整数的数个数。

首先，如果 $sumTop-sumA$ 足够大，有足够多的非整数可以让 $sumTop$ 一直减，那么肯定一直减会优（能迈大步的情况下迈大步总比迈小步或者停在原地要优）。也就是如果 $sumTop-sumA>\min(len,sumNInt)$（$\min$ 就是说要是不是整数的数足够多可以一直减，那么必须有一个上界。因为毕竟只能更改 $len$ 个元素使其变成下取整），那么直接让 $sumTop$ 减去 $sumA + \min(len,sumNInt)$ 就是答案。

但是如果 $sumTop$ 和 $sumA$ 的差别没有那么大，那么就要考虑让一些整数参与进来，也就是让 $sumTop$ 不变（防止减得过多反而不优）。首先要把 $sumTop$ 减到和 $sumA$ 的差的绝对值不超过 $0.5$，也就是加一减一都不会更优。然后看还剩下多少个数需要参与进来（毕竟必须要让 $len$ 个下取整）。

剩下的数如果有整数那就尽可能放吧。因为整数不影响结果，还是很优。但是如果没有整数的话，只能让本来已经很优的情况变坏，也就是把剩下的需要减掉的 $1$ 全部减掉。

具体看代码吧。

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  int f = 1;
  while (ch = getchar(), !isdigit(ch)) {
    if (ch == '-') f = -1;
  }
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) { x = x * 10 + (ch ^ 48); }
  x *= f;
}

const int N = 2e6 + 10;
int n, m, sumNInt, nInt[N];
double a, top, sumA, A[N], sumTop, Top[N];
/* 有sum的对应为相应前缀和的区间和（因为在区间内操作）
nInt: 不是整数的数的个数的前缀和
A: 题中原序列的前缀和
Top: 题中原序列上取整的前缀和
*/
inline void main() {
  read(n), read(m);
  for (int i = 1; i <= n; ++i) {
    scanf("%lf", &a), top = ceil(a);
    A[i] = A[i - 1] + a;
    Top[i] = Top[i - 1] + top;
    nInt[i] = nInt[i - 1] + (top != (int)a);
  }
  for (int i = 1, l, r, len, v, t; i <= m; ++i) {
    read(l), read(r), len = (r - l + 1) / 2;
    sumA = A[r] - A[l - 1];
    sumTop = Top[r] - Top[l - 1];
    sumNInt = nInt[r] - nInt[l - 1];
    if (sumTop - min(sumNInt, len) > sumA) { // 如果一直减一能一直变优
      sumTop -= min(sumNInt, len); // 减掉这么多数（非整数不够了用整数补齐（就是答案不动））
    } else {
      v = min(min(sumNInt, len), int(sumTop - sumA + 0.5)); // 能动用的非整数让答案最优
      /* 
      这个0.5的作用可以思考一下，先从举例子开始，比如sumA=1.4和sumTop=2（原序列可以是0.4,1）。
      sumTop-sumA是0.6，下取整就没了，但是显然sumTop减一个1以后差的绝对值变成了0.4。
      也就是 让0.4下取整的同时1上取整 而非 让0.4上取整的同时让1下取整。
      所以要充分利用条件，多手玩几个例子就好了qwq。
      */
      t = len - v; // 剩下只能让答案不变或更劣的数
      sumTop -= v; // 先最优，不管那么多
      if (len * 2 - sumNInt < t) { // 如果整数数量小于还需要的数（不小于的话答案就不用变了）
        sumTop -= t - (len * 2 - sumNInt); // 那就需要减去这么多的权值让答案更劣（毕竟为了符合题意）
      }
    }
    printf("%.3lf\n", abs(sumTop - sumA)); // 输出最优的答案
  }
}

}  // namespace STDquantum
```
