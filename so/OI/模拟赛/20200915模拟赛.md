# 购物

## 题目描述

X 城的商场中，有着琳琅满目的各种商品。一日，小X 带着小Y 前来购物，小Y 一共看中了n件商品，每一件商品价格为Pi。小X 现在手中共有m个单位的现金，以及k 张优惠券。小X 可以在购买某件商品时，使用至多一张优惠券，若如此做，该商品的价格会下降至Qi。

小X 希望尽可能多地满足小Y 的愿望，所以小X 想要知道他至多能购买多少件商品。

## 输入输出格式

### 输入格式

第一行包含三个整数n,k,m，表示商品总数，小X 拥有的优惠券与现金总数。

接下来n行每行包含两个整数Pi,Qi。

### 输出格式

共一行包含一个整数，表示小X 至多能购买的物品数。

## 输入输出样例

### 输入样例 #1

```
4 1 7
3 2
2 2
8 1
4 3
```

### 输出样例 #1

```
3
```

## 说明

样例解释：一种最优的购买方式是购买1,2,3号物品，并用优惠券购买物品3，总共花费为3+2+1=6。 

## 题解

显然是个贪心，在luogu还有原题（[P3045 [USACO12FEB]Cow Coupons G](https://www.luogu.com.cn/problem/P3045)）。

（模拟赛数据水到什么奇奇怪怪的贪心都能拿不少分）

设三个优先队列，一个用来放 $p$，一个用来放 $q$，还有一个用作反悔，放已经选了的 $q$ 的 $p-q$。

啥意思呢？每次取的时候，比较反悔堆堆顶加上 $q$ 堆堆顶的值和 $p$ 堆堆顶的值，哪个小取哪个。

$p$ 小容易理解，不用券选就完了。

那个玩意呢？

我们设反悔堆堆顶里存的 $p-q$ 对应的那个元素是 $x$，$q$ 的堆顶对应的那个是 $y$。那么我们这两次取的代价和是 $q_x+(p_x-q_x)+q_y$，也就是把 $x$ 的券给 $y$ 用。相当于做了一次返回操作。

那么为啥反悔完了要取 $x$ 的 $p$ 呢？为啥不会出现 $x$ 的 $p$ 很大不优了的情况呢？

我不会证，洛谷copy：

> **基本原则：** 
>
> $k$ 张优惠券不用白不用。
>
> 我们首先按照奶牛的折扣价 $C_i$ 从小到大排序。
>
> 排完序后前 $k$ 头牛必定在答案当中，但不一定会在它们身上用上优惠券。
>
> **证明(反证法):**  
>
> 如果第 $i$ ( $i$ $\leq$ $k$ ) 头牛不在答案当中，那么 $k$ 张优惠券就无法全部用完，只能用 $k-x$ 张，这样会剩 $x$ 张。假设目前剩了 $1$ 张，这 $1$ 张根据“基本原则”，还是一定会用的，那么一定会用在 $j$ ( $j>k$ )身上。而用在 $j$ 身上，不如用在 $i$ 身上划算。所以不能这样做。至此证明了前 $k$ 头牛必定在答案当中。
>
> 接下来证明不一定会在前 $k$ 头牛上用优惠券。由于 $C_i+P_j>C_j+P_i$ 的情况是有可能的，所以优惠券不一定会全部用在前 $k$ 头牛上。如果说 $i$ 是原价购买，优惠券仍然必然会用在 $j$ 上，这也不和前面的证明和原则冲突。综上所述，前 $k$ 头牛不一定会用上优惠券，但一定在答案中。

这个算法也同理，在取的时候一定会涉及到前 $k$ 小的，所以反悔操作是对的。

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  while (ch = getchar(), !isdigit(ch)) {}
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) x = x * 10 + (ch ^ 48);
}

typedef long long ll;
const int N = 2e6 + 10;

struct E {
  ll x;
  int id;
  inline friend bool operator<(const E &a, const E &b) {
    return a.x > b.x;
  }
};

int n, k, p[N], q[N];
ll m;
priority_queue<E> q1, q2;
priority_queue<ll, vector<ll>, greater<ll> > e;
bool vis[N];

inline void pushup() {
  while (vis[q1.top().id]) q1.pop();
  while (vis[q2.top().id]) q2.pop();
}

inline void main() {
  read(n), read(k), read(m);
  for (int i = 1; i <= n; ++i) {
    read(p[i]), read(q[i]);
    q1.push((E){p[i], i}), q2.push((E){q[i], i});
    if (i <= k) e.push(0ll);
  }
  int ans = 0;
  for (E tmp; m > 0 && ans < n; ++ans) { // m>0和ans<n不能取等！
    pushup();
    if (e.top() + q2.top().x < q1.top().x) {
      tmp = q2.top(), q2.pop();
      m -= e.top() + tmp.x, vis[tmp.id] = true;
      e.pop(), e.push(p[tmp.id] - q[tmp.id]);
    } else {
      tmp = q1.top(), q1.pop();
      m -= tmp.x, vis[tmp.id] = true;
    }
    if (m < 0) break;
  }
  printf("%d", ans);
}

}  // namespace STDquantum

int main() {

  freopen("D:\\OI\\STDquantum\\test.in", "r", stdin);
  freopen("D:\\OI\\STDquantum\\test.out", "w", stdout);
#endif
#ifndef LOCAL
  freopen("shopping.in", "r", stdin);
  freopen("shopping.out", "w", stdout);
#endif
#endif
  STDquantum::main();
  return 0;
}
```

# 拆网线

## 题目描述

企鹅国的网吧们之间由网线互相连接，形成一棵树的结构。现在由于冬天到了，供暖部门缺少燃料，于是他们决定去拆一些网线来做燃料。但是现在有K只企鹅要上网和别人联机游戏，所以他们需要把这K只企鹅安排到不同的机房（两只企鹅在同一个机房会吵架），然后拆掉一些网线，但是需要保证每只企鹅至少还能通过留下来的网线和至少另一只企鹅联机游戏。 所以他们想知道，最少需要保留多少根网线？

## 输入输出格式

### 输入格式

第一行一个整数T，表示数据组数；

每组数据第一行两个整数N，K，表示总共的机房数目和企鹅数目。

第二行N-1个整数，第i个整数Ai表示机房i+1和机房Ai有一根网线连接(1≤Ai≤i)。

### 输出格式

每组数据输出一个整数表示最少保留的网线数目。

## 输入输出样例

### 输入样例 #1

```
2
4 4
1 2 3
4 3
1 1 1
```

### 输出样例 #1

```
2
2
```

## 说明

对于30%的数据：$N≤15$； 

对于50%的数据：$N≤300$； 

对于70%的数据：$N≤2000$； 

对于100%的数据：$2≤K≤N≤100000,T≤10$。

## 题解

可以发现最大可能的答案是 $K-1$，也就是全在一个连通块里，最小可能的答案是两个两个分别联通。

因此得出结论：要寻找树中最多的大小为 $2$ 的连通块数，剩下的多一只企鹅多一条边。

设 $f_{i,0/1}$ 代表以 $i$ 为根的子树中大小为 $2$ 的最多连通块数，$0$ 代表 $i$ 点不选，$1$ 代表 $i$ 点选。

那么有转移 $\displaystyle f_{i,0}=\sum_{v\in son_i}\max\{f_{v,0},f_{v,1}\}$。意思是如果 $i$ 不选，那么 $v$ 选不选都无所谓，挑最大的加就好了。实际上这个地方选 $f_{v,1}$ 是一定比 $f_{v,0}$ 优的，证明如下：

在 $f_{v,0}$ 最大的情况时，$v$ 的所有子树都被填满了，这时候 $f_{v,1}$ 是等于 $f_{v,0}$ 的（随便把子节点的一个匹配点给 $v$）。而如果 $v$ 的子树没有被填满，那么至少会有一个空位，这个空位的安排会有两种情况：

{% asset_image 1.png %}

- $v$ 利用不了。

  如图一情况，那么 $f_{v,1}=f_{v,0}$。

- $v$ 能利用。

  如图二情况，这种情况能空出一个 $v$ 的子节点给 $v$ 匹配，那么 $f_{v,1}=f_{v,0}+1$。

则 $f_{v,1}\ge f_{v,0}$，证毕。

所以 $\displaystyle f_{i,0}=\sum_{v\in son_i}f_{v,1}$。

因此，我们能得出 $f_{i,1}$，选出 $f_{v,1}$ 最小的一个节点（设为 $s$）贡献变成 $f_{s,0}$，剩下的贡献是 $f_{v,1}$。

所以 $\displaystyle f_{i,1}=f_{s,0}+1+\sum_{v\in son_i\wedge v\neq s}f_{v,1}$。

其实不用这么复杂，就 $f_{i,1}=\max\{f_{i,0}-f_{v,1}+f_{v,0}+1\}$ 就好了，不解释了，前面解释够多了。

嗯。。可以了。

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  while (ch = getchar(), !isdigit(ch)) {}
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) x = x * 10 + (ch ^ 48);
}

const int N = 2e5 + 10;

int tot, head[N];
struct E {
  int next, to;
} e[N];
inline void add(int x, int y) {
  e[++tot] = (E){head[x], y}, head[x] = tot;
  e[++tot] = (E){head[y], x}, head[y] = tot;
}

int f[N][2];
void dfs(int s, int fa) {
  for (int i = head[s], v; i; i = e[i].next) {
    if ((v = e[i].to) == fa) continue;
    dfs(v, s), f[s][0] += f[v][1];
  }
  for (int i = head[s], v; i; i = e[i].next) {
    if ((v = e[i].to) == fa) continue;
    f[s][1] = max(f[s][1], f[s][0] - f[v][1] + f[v][0] + 1);
  }
}

int n, k, ans;
inline void main() {
  int T;
  for (read(T); T--;) {
    tot = 0, memset(head, 0, sizeof(head));
    memset(f, 0, sizeof(f));
    read(n), read(k);
    for (int i = 1, x; i < n; ++i) { read(x), add(x, i + 1); }
    dfs(1, 0), ans = f[1][1];
    if (ans * 2 < k) {
      ans += k - ans * 2;
    } else {
      ans = k / 2 + k % 2;
    }
    printf("%d\n", ans);
  }
}

}  // namespace STDquantum

int main() {

#ifdef LOCAL
  freopen("D:\\OI\\STDquantum\\test.in", "r", stdin);
  freopen("D:\\OI\\STDquantum\\test.out", "w", stdout);
#endif
#ifndef LOCAL
  freopen("tree.in", "r", stdin);
  freopen("tree.out", "w", stdout);
#endif
#endif
  STDquantum::main();
  return 0;
}
```

# 密室

## 题目描述

小X 正困在一个密室里，他希望尽快逃出密室。

密室中有N 个房间，初始时，小X 在1 号房间，而出口在N 号房间。

密室的每一个房间中可能有着一些钥匙和一些传送门，一个传送门会单向地创造一条从房间X 到房间Y 的通道。另外，想要通过某个传送门，就必须具备一些种类的钥匙（每种钥匙都要有才能通过）。幸运的是，钥匙在打开传送门的封印后，并不会消失。

然而，通过密室的传送门需要耗费大量的时间，因此，小X 希望通过尽可能少的传送门到达出口，你能告诉小X 这个数值吗？

另外，小X 有可能不能逃出这个密室，如果是这样，请输出"No Solution"。

## 输入输出格式

### 输入格式

第一行三个整数N,M,K，分别表示房间的数量、传送门的数量以及钥匙的种类数。

接下来N 行，每行K 个0 或1，若第i 个数为1，则表示该房间内有第i 种钥匙，若第i 个数为0，则表示该房间内没有第i 种钥匙。

接下来M 行，每行先读入两个整数X,Y，表示该传送门是建立在X 号房间，通向Y 号房间的，再读入K 个0 或1，若第i 个数为1，则表示通过该传送门需要i 种钥匙，若第i 个数为0，则表示通过该传送门不需要第i 种钥匙。

### 输出格式

输出一行一个“No Solution”，或一个整数，表示最少通过的传送门数。

## 输入输出样例

### 输入样例 #1

```
3 3 2
1 0
0 1
0 0
1 3 1 1
1 2 1 0
2 3 1 1
```

### 输出样例 #1

```
2
```

## 题解

发现每次往外扩展的时候能走到哪个房间是确定的（因为有多少钥匙是固定的）。

而 $k$ 最大只有 $10$，$n$ 最大只有 $5000$！

每个路径的状态都是固定的，所以直接考虑把状态和点的编号组合在一起，跑一个边权全是一的spfa就好了。

然而这题数据并没环。所以直接bfs即可。（加个vis还是更保险）

如果有边权的话，就是一个分层图最短路。

```cpp
#include <bits/stdc++.h>
using namespace std;

namespace STDquantum {

template <typename T> inline void read(T &x) {
  char ch;
  while (ch = getchar(), !isdigit(ch)) {}
  x = (ch ^ 48);
  while (ch = getchar(), isdigit(ch)) x = x * 10 + (ch ^ 48);
}

const int N = 6e3 + 10, M = 1 << 10, inf = 0x3f3f3f3f;
int tot, head[N];
struct E {
  int next, to, w;
} e[N];
inline void add(int x, int y, int z) {
  e[++tot] = (E){head[x], y, z}, head[x] = tot;
}

int n, m, k, sta[N];

struct T {
  int x, sta;
};

int dis[M][N];
bool vis[M][N];
void bfs() {
  memset(dis, 0x3f, sizeof(dis));
  queue<T> q;
  dis[sta[1]][1] = 0, q.push((T){1, sta[1]});
  while (!q.empty()) {
    int u = q.front().x, s = q.front().sta;
    q.pop(), vis[s][u] = false;
    for (int i = head[u], v, s1; i; i = e[i].next) {
      v = e[i].to, s1 = s | sta[v];
      if (((e[i].w & s) == e[i].w) && dis[s1][v] > dis[s][u] + 1) {
        dis[s1][v] = dis[s][u] + 1;
        if (!vis[s1][v]) q.push((T){v, s1}), vis[s1][v] = true;
      }
    }
  }
}

inline void main() {
  read(n), read(m), read(k);
  for (int i = 1, s1; i <= n; ++i) {
    for (int j = 0; j < k; ++j) read(s1), sta[i] |= (s1 << j);
  }
  for (int i = 1, s, s1, x, y; i <= m; ++i) {
    read(x), read(y), s = 0;
    for (int j = 0; j < k; ++j) read(s1), s |= (s1 << j);
    add(x, y, s);
  }
  bfs();
  int ans = inf;
  for (int i = 0; i < (1 << k); ++i) ans = min(ans, dis[i][n]);
  if (ans == inf) {
    puts("No Solution");
  } else {
    printf("%d", ans);
  }
}

}  // namespace STDquantum

int main() {
#ifndef ONLINE_JUDGE
#ifdef LOCAL
  freopen("D:\\OI\\STDquantum\\test.in", "r", stdin);
  freopen("D:\\OI\\STDquantum\\test.out", "w", stdout);
#endif
#ifndef LOCAL
  freopen("room.in", "r", stdin);
  freopen("room.out", "w", stdout);
#endif
#endif
  STDquantum::main();
  return 0;
}
```
